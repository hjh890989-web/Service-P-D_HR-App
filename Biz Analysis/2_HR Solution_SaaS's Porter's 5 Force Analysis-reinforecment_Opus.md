# Part 1: 시장 정의 적절성 검증

## 1.1 현재 시장 정의 재확인

기준 문서에서 정의한 시장:

> **"중소/중견기업 대상 Workforce Risk Management 시장 (HR + EHS + Compliance 통합 영역)"**  
> — HR SaaS, EHS 시스템, ERP, 노무사/안전컨설팅, 수기관리(엑셀 등)를 모두 포함한 "인력 리스크 관리"의 넓은 시장

---

## 1.2 시장 정의 적절성 평가

### 평가 매트릭스

| 평가 기준 | 판정 | 근거 |
|-----------|------|------|
| **고객 관점 일관성** (같은 문제를 해결하려는 고객군인가?) | ✅ 적절 | 중소/중견기업의 경영자는 "노동법 위반 리스크"와 "산업재해 리스크"를 **하나의 경영 리스크**로 인식함. 별개의 문제로 인식하는 것은 솔루션 공급자의 관점 |
| **예산 경합성** (고객의 같은 예산 내에서 경합하는가?) | ✅ 적절 | 중소기업의 "관리비/외주비" 예산에서 노무사비, 안전컨설팅비, SaaS 이용료, ERP 유지보수비가 동일 예산 풀에서 경합함 |
| **기능적 대체 가능성** (서로 대체할 수 있는가?) | ⚠️ 부분 적절 | HR SaaS와 노무사는 직접 대체 관계. 그러나 IoT 기반 현장 센서 시스템은 기능적으로 다른 영역 → 경계 설정 필요 |
| **규제 공통성** (같은 법적 프레임워크에 속하는가?) | ✅ 적절 | 근로기준법 + 산업안전보건법 + 중대재해처벌법이 공통 규제 프레임워크. 환경(E) 영역은 별도 법체계 |
| **TAM/SAM/SOM 확장 가능성** | ✅ 적절 | 넓은 정의에서 시작하여 세그먼트별로 축소 가능. 시장 규모 추정의 기반이 됨 |

### 종합 판정

> **현재 정의는 "대체로 적절"하나, 3가지 경계 조건을 명확히 해야 한다.**

## 1.3 포함되어야 할 산업/서비스 범위

### A. 반드시 포함 (Core Market)

| 범주 | 포함 대상 | 포함 이유 |
|------|-----------|-----------|
| **HR SaaS** | flex, 시프티, ZUZU, 페어리HR 등 | 근태·급여·계약 관리 = 노동법 컴플라이언스의 핵심 도구. 직접 경쟁 |
| **EHS 솔루션** | 세이프로, 안전하이, 이지세이프, 스마플 등 | 산업안전보건법·중대재해처벌법 대응 도구. 직접 경쟁 |
| **ERP 인사/안전 모듈** | 더존 iCUBE/아마란스, 영림원 K-System 등 | 기존 ERP의 인사·안전관리 모듈이 통합 관리 기능 제공. 대체재이자 잠재 경쟁자 |
| **노무사 자문 서비스** | 공인노무사 사무소, 노무법인 | 법적 리스크 관리의 전통적 방식. 가장 강력한 인적 대체재 |
| **안전보건 컨설팅** | 민간 안전컨설팅 업체, 안전보건공단 지원사업 수행기관 | 안전관리체계 구축 = 고객의 동일 문제 해결 수단 |
| **수기/엑셀 관리** | 현재 70%+ 기업의 현실적 관리 방식 | 가장 큰 "비소비(Non-consumption)" 시장. 실질적 최대 경쟁자 |
| **급여 아웃소싱** | 급여 대행 서비스(노무법인, 급여 전문 업체) | 인사관리의 핵심 기능을 외주화한 형태. 예산 경합 |
| **교육 관리 플랫폼** | 법정의무교육 플랫폼(산업안전교육, 성희롱예방교육 등) | 법적 교육의무 관리 = 컴플라이언스의 일부 |

### B. 포함 권장 (Adjacent Market — TAM 확장 시 필요)

| 범주 | 포함 대상 | 포함 이유 | 주의사항 |
|------|-----------|-----------|----------|
| **보험 연계 서비스** | 산재보험 관련 서비스, 배상책임보험 | 리스크의 금융적 관리 수단. 통합 시 가치 제안 강화 | SAM 단계에서 분리 |
| **근로자 건강관리** | 특수건강검진 관리, 직업병 예방 서비스 | 산업안전보건법상 의무. EHS의 H(Health) 영역 | MVP에서는 제외 가능 |
| **출입/현장 관리** | 현장 출입관리 시스템, 방문자 관리 | 건설현장 원/하청 인력관리와 직결 | 하드웨어 연동 복잡성 고려 |
| **정부 지원사업 연계** | 스마트공장, 클린사업장, 안전장비 지원사업 | 고객 도입 비용 보조의 핵심 채널 | 시장 규모 추정 시 별도 처리 |

### C. 제외해야 할 범위

| 범주 | 제외 대상 | 제외 이유 |
|------|-----------|-----------|
| **환경(Environment) 관리** | 폐기물 관리, 대기/수질 배출 관리, 탄소 배출 관리 | 별도 법체계(환경보호법), 별도 시장, 별도 전문가 영역. EHS의 "E"이지만 고객 페르소나·예산·의사결정이 다름 |
| **대기업 솔루션** | SAP EHS, Oracle HCM, ServiceNow HSE | 중소/중견기업 시장과 고객·가격·세일즈 모델이 완전히 다름. TAM에는 참조하되 주요 분석 대상은 아님 |
| **IoT 하드웨어** | 가스 감지 센서, 스마트 안전모, CCTV 장비 자체 | 하드웨어 시장은 별도 시장. 소프트웨어와 "연동"은 가능하나 시장 정의에 포함 시 범위 과도 확장 |
| **채용/헤드헌팅** | 채용 플랫폼(사람인, 잡코리아), 헤드헌팅 | "인력 확보"는 "인력 리스크 관리"와 다른 고객 문제. 예산 풀도 다름 |
| **CRM/세일즈 도구** | HubSpot, Salesforce 등 | 인력 관리와 무관한 비즈니스 도구 |
| **순수 교육 LMS** | 일반 이러닝 플랫폼 (법정의무교육 이외) | 자기계발/역량 교육은 컴플라이언스와 무관 |

## 1.4 시장 정의 수정안

### 현재 정의 (유지)

> **"중소/중견기업 대상 Workforce Risk Management 시장"**

### 보완된 해석 가이드 (추가 제안)

현재 정의는 적절하나, 후속 분석(TAM/SAM/SOM) 시 **3개 레이어로 분해하여 사용**할 것을 권장:

```
┌─────────────────────────────────────────────────────────┐
│                     TAM Layer                            │
│  "Workforce Risk Management" 전체                        │
│  (HR + EHS + Compliance + 교육 + 보험연계 + 건강관리)    │
│  - 대기업 포함, 모든 솔루션/서비스 유형 포함             │
│                                                          │
│  ┌───────────────────────────────────────────────────┐   │
│  │                  SAM Layer                         │   │
│  │  중소/중견기업(20~300인) 대상                     │   │
│  │  HR + EHS + Compliance 통합 관리                  │   │
│  │  (SaaS + 노무사 + 컨설팅 + 수기관리 포함)        │   │
│  │                                                    │   │
│  │  ┌─────────────────────────────────────────────┐  │   │
│  │  │               SOM Layer                      │  │   │
│  │  │  제조/건설/물류 (20~200인)                   │  │   │
│  │  │  SaaS 도입 가능 세그먼트                     │  │   │
│  │  │  (산재 경험 / 노동청 점검 경험 기업)         │  │   │
│  │  └─────────────────────────────────────────────┘  │   │
│  └───────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 경계 조건 3가지 (반드시 명시할 것)

| # | 경계 조건 | 이유 |
|---|-----------|------|
| 1 | **환경(E)은 제외한다** | EHS의 E(Environment)는 별도 법체계, 별도 전문가, 별도 예산. 포함하면 시장이 과도하게 넓어져 분석 주의(focus)가 흐려짐 |
| 2 | **하드웨어(IoT 센서/장비)는 제외하되, 연동 가능성은 인정한다** | 하드웨어 시장 포함 시 시장 규모 추정이 왜곡됨. 그러나 소프트웨어-하드웨어 연동은 미래 확장 시나리오로 유지 |
| 3 | **채용/인재확보 영역은 제외한다** | 채용은 "리스크 관리"가 아니라 "리소스 확보". 고객의 문제 인식(Job-to-be-Done)이 본질적으로 다름 |

## 1.5 시장 정의 적절성 최종 평가

| 항목 | 평가 |
|------|------|
| 너무 넓은가? | ⚠️ **약간 넓음** — 환경(E) 관리, IoT 하드웨어, 채용 영역이 명시적으로 경계 지어지지 않으면 혼란 유발 |
| 너무 좁은가? | ✅ **아님** — 교육 관리, 급여 아웃소싱, 수기관리까지 포함하여 충분히 넓은 범위 확보 |
| TAM/SAM/SOM 연계 가능한가? | ✅ **가능** — 3개 레이어 분해 모델 적용 시 자연스럽게 확장/축소 가능 |

> **최종 판정: 현재 시장 정의를 유지하되, 위 3가지 경계 조건을 명시적으로 부가하라.**

# Part 2: 대체재 심층 분석

## 2.0 분석 프레임워크

기준 문서의 Force 3(대체재의 위협)에서 대체재를 맵핑하였다. 본 섹션에서는 **4대 핵심 대체재**를 실무 수준에서 심층 분석한다.

분석 대상:
1. 노무사 (공인노무사 자문/대행)
2. 안전관리 컨설팅 (민간 + 정부 지원)
3. ERP (더존 등 인사/안전 모듈)
4. 엑셀/수기관리

---

## 2.1 노무사 (공인노무사 자문/대행 서비스)

### 2.1.1 실제 사용 방식

| 서비스 유형 | 설명 | 이용 빈도 |
|------------|------|-----------|
| **월 정액 고문 계약** | 월 10~50만 원 수준으로 상시 전화/메일 자문. 취업규칙, 근로계약서 검토, 노동법 QnA | 가장 보편적 |
| **급여 아웃소싱 (Payroll Outsourcing)** | 급여 계산, 4대보험 신고, 연말정산을 노무법인이 대행. 고문 계약과 패키지 | 50인 미만에서 빈번 |
| **사건 대리 (별도 수임)** | 부당해고 구제, 임금체불, 노동청 진정/고발 대응. 착수금 + 성공보수 별도 | 사건 발생 시 |
| **취업규칙/인사규정 정비** | 연 1~2회 규정 전면 검토 및 개정. 프로젝트 단위 별도 과금 | 법 개정 시 |
| **안전보건관리체계 구축 자문** | 중대재해처벌법 대응을 위한 체계 구축. 최근 급증하는 서비스 영역 | 법 확대 이후 급증 |

**고객의 실제 이용 패턴 (시나리오):**
```
[일반적 중소기업 - 50인 제조업]

월요일: 직원이 야근수당 질문 → 관리부장이 노무사에 카톡/전화
화요일: 노무사 답변 (1~2시간 소요)
목요일: 노동청에서 근로감독 통보 → 노무사에 긴급 연락
금요일: 노무사 사무실 방문 → 대응 서류 작성 시작 (3~5일 소요)
월말: 노무사에게 급여 데이터(엑셀) 전달 → 노무사가 급여 계산 → 관리부장 확인 → 지급
```

### 2.1.2 고객이 노무사를 선택하는 이유

| 선택 이유 | 설명 | 강도 |
|-----------|------|------|
| **법적 안전감 (책임 이전 심리)** | "전문가에게 맡겼으니 문제 생겨도 내 잘못은 아니다"라는 심리적 보험 | 🔴 매우 강함 |
| **관계 기반 신뢰** | "우리 회사 사정을 아는 노무사"에 대한 관계적 신뢰. 전환 비용이 심리적으로 매우 높음 | 🔴 매우 강함 |
| **유연한 대응** | 법에 없는 "그레이존" 질문에도 경험 기반 답변 가능. SaaS의 규칙 기반 답변과 차원이 다름 | 🟡 강함 |
| **초기 비용 제로** | "문제 생기면 그때 부르면 된다" → 고문 계약 없이도 건별 이용 가능 | 🟡 강함 |
| **IT 학습 불필요** | 새로운 시스템을 배울 필요 없음. 전화 한 통이면 해결 | 🟡 강함 |

### 2.1.3 노무사의 한계 (SaaS 관점의 기회)

| 한계 | 구체적 상황 | SaaS가 해결할 수 있는가? |
|------|------------|------------------------|
| **비실시간 대응** | 급한 질문에 답변까지 평균 4~24시간. 야간/주말 대응 불가 | ✅ SaaS는 24/7 자동 안내 가능 |
| **확장 불가능성** | 노무사 1명이 동시에 관리 가능한 사업장 수 한계 (약 20~30곳) | ✅ SaaS는 무제한 확장 |
| **데이터 부재** | 상담 내역·서류가 노무사 개인에 귀속. 기업이 노무사를 바꾸면 히스토리 소실 | ✅ SaaS는 데이터 축적·이관 가능 |
| **사전 예방 불가** | 문제 발생 후 대응(Reactive). "3개월 후 발생할 위반 사항"을 미리 알려주지 않음 | ✅ SaaS는 사전 리스크 탐지 가능 |
| **비용 불투명** | 사건별 추가 비용이 예측 불가. "노동청 대응"만으로 수백만 원 청구 가능 | ✅ SaaS는 월 정액 예측 가능 비용 |
| **산업안전 전문성 제한** | 대부분의 노무사는 노동법 전문. 산업안전보건법·중대재해처벌법은 별도 전문성 영역 | ✅ SaaS는 HR + EHS 통합 가능 |

## 2.2 안전관리 컨설팅

### 2.2.1 실제 사용 방식

| 서비스 유형 | 제공 주체 | 비용 | 기간 | 특징 |
|------------|-----------|------|------|------|
| **정부 무상 컨설팅** | 안전보건공단 + 수행기관 (노무법인, 컨설팅사) | 전액 정부 부담 (무료) | 3~6개월 | 5인 이상 50인 미만 우선. 연간 예산 한도 내 모집 |
| **민간 유료 컨설팅** | 안전컨설팅 법인, 기술사 사무소 | 200~1,000만 원 (규모·업종별 상이) | 1~3개월 | M/D(Man-Day) 기반 과금. 사업장 규모·위험도에 따라 산정 |
| **안전관리 대행** | 안전관리 전문기관 | 월 50~150만 원 | 상시 계약 | 산업안전보건법상 안전관리자 선임 의무를 외부 위탁 |
| **위험성평가 대행** | 위험성평가 전문업체 | 100~500만 원 (건당) | 1~4주 | 법적 의무인 위험성평가를 대행. 연 1회+ 갱신 |

**고객의 실제 이용 패턴 (시나리오):**
```
[50인 제조업 - 정부 무상 컨설팅 시나리오]

1월: 중대재해처벌법 적용 사실 인지 → 안전보건공단 홈피에서 무상 컨설팅 신청
3월: 수행기관(컨설팅사) 배정 → 현장 방문 1회차 (현황 파악)
4월: 위험성평가 실시, 안전보건관리체계 매뉴얼 초안 작성
5월: 매뉴얼 전달 + 교육 1회
6월: 사업 종료. 수행기관 철수.

→ 이후: 매뉴얼은 서랍에 보관. 현장은 이전과 동일. 
→ 1년 후 노동청 점검 시: "서류는 있는데 실제 이행 증거가 없다" 지적
```

### 2.2.2 고객이 안전컨설팅을 선택하는 이유

| 선택 이유 | 설명 | 강도 |
|-----------|------|------|
| **무료 (정부 지원)** | "공짜니까 일단 받자". 중소기업의 최대 동기 | 🔴 매우 강함 |
| **법적 의무 면피** | "컨설팅 받았다"는 사실 자체가 법적 방어 논거가 됨 | 🔴 매우 강함 |
| **일회성 해결** | 지속적 비용 발생 없음. "한 번 하고 끝" 심리 | 🟡 강함 |
| **전문가 브랜드** | 안전보건공단·대한산업안전협회 명의의 신뢰 | 🟡 중간 |

### 2.2.3 안전컨설팅의 한계 (SaaS 관점의 기회)

| 한계 | 구체적 상황 | SaaS가 해결할 수 있는가? |
|------|------------|------------------------|
| **일회성 (사후관리 부재)** | 컨설팅 종료 후 이행 여부를 아무도 모니터링하지 않음 | ✅ SaaS는 상시 모니터링·알림 가능 |
| **서류 중심** | 매뉴얼·체크리스트를 "종이로" 전달. 현장에서 활용되지 않음 | ✅ SaaS는 모바일 기반 현장 활용 가능 |
| **정부 예산 소진 리스크** | 무상 컨설팅은 연간 예산 내 선착순. 신청 못 하면 대기 | ✅ SaaS는 즉시 도입 가능 |
| **획일적 접근** | 업종·규모 상관없이 비슷한 매뉴얼 제공. 맞춤형 아님 | ✅ SaaS는 업종·규모별 커스터마이징 가능 |
| **데이터 미축적** | 컨설팅 결과가 기업 내부 시스템에 축적되지 않음 | ✅ SaaS는 이력 기반 리스크 프로파일 구축 |

## 2.3 ERP (인사/안전 모듈)

### 2.3.1 실제 사용 방식

| ERP 유형 | 대표 제품 | 인사 모듈 범위 | 안전관리 범위 | 도입 비용 (추정) |
|---------|-----------|--------------|-------------|----------------|
| **국산 중소기업용** | 더존 iCUBE/아마란스 10 | 급여, 인사정보, 근태, 4대보험 | ❌ 거의 없음 | 초기 500~3,000만 원 + 연 유지비 10~20% |
| **국산 중견기업용** | 영림원 K-System | 급여, 인사, 근태, 인사평가 | ⚠️ 제한적 (안전점검 정도) | 초기 2,000~5,000만 원 + 연 유지비 |
| **글로벌** | SAP S/4HANA | HR 전체 + EHS 모듈 | ✅ 풀스택 EHS | 억 단위 → 중소기업 비현실적 |

**고객의 실제 이용 패턴 (시나리오):**
```
[80인 제조업 - 더존 iCUBE 사용 시나리오]

급여일(매월 25일):
- 관리부 직원이 더존 ERP에서 급여 계산 실행
- 4대보험·소득세 자동 계산 → 은행 이체 연동
- 근태 데이터는 별도 엑셀로 관리 후 더존에 수동 입력

안전관리:
- 더존 ERP에 안전 기능 없음
- 안전관리자(겸직)가 별도 엑셀로 위험성평가 서류 관리
- 안전교육 이수 현황은 종이 대장에 기록
- 사고 발생 시 보고서는 워드파일로 작성 → 노무사에 전달

→ 결과: 인사 데이터와 안전 데이터가 완전히 분리되어 있음
→ 직원 A의 근태이상(잦은 결근) + 현장 아차사고 이력을 연결할 방법 없음
```

### 2.3.2 고객이 ERP를 선택하는 이유

| 선택 이유 | 설명 | 강도 |
|-----------|------|------|
| **회계와의 통합** | "급여 → 회계 전표 자동 연동"이 ERP의 최대 강점. 이것 때문에 ERP를 버리지 못함 | 🔴 매우 강함 |
| **이미 도입했으니 추가 비용 회피** | "이미 더존 쓰고 있는데 인사모듈도 있으니 별도 SaaS는 불필요" | 🔴 매우 강함 |
| **세무사/회계사의 권유** | 세무사가 더존을 쓰므로 데이터 호환을 위해 기업도 더존 사용 | 🟡 강함 |
| **정부 지원** | 스마트공장 바우처, 클라우드 바우처 등으로 ERP 도입 비용 보조 | 🟡 중간 |

### 2.3.3 ERP의 한계 (SaaS 관점의 기회)

| 한계 | 구체적 상황 | SaaS가 해결할 수 있는가? |
|------|------------|------------------------|
| **안전관리 기능 부재** | 더존/영림원 모두 중대재해처벌법 대응 기능 없음 | ✅ 핵심 차별화 포인트 |
| **모바일/현장 UX 취약** | ERP는 PC 사무실 중심 설계. 현장 작업자가 쓸 수 없음 | ✅ 모바일 퍼스트 설계 가능 |
| **법 개정 반영 지체** | 법 개정 시 ERP 업데이트까지 수개월~1년 소요 | ✅ SaaS는 즉시 반영 가능 |
| **커스터마이징 비용** | 추가 기능 요청 시 수백만 원 개발비 발생 | ✅ SaaS는 표준 기능으로 커버 |
| **Lock-in 강도 높음** | 급여+회계+세무 연동 → 전환 시 전체 업무 프로세스 변경 필요 | ⚠️ 병행 사용(Coexist) 전략 필요 |

## 2.4 엑셀/수기관리

### 2.4.1 실제 사용 방식

| 관리 영역 | 엑셀 사용 방식 | 사용 빈도 |
|-----------|---------------|-----------|
| **근태관리** | 출퇴근 시간을 엑셀에 수동 입력. 또는 카드키/지문 출입 데이터를 엑셀로 다운로드 | 매일 |
| **급여계산** | 엑셀 수식으로 기본급+초과근무수당+공제액 계산. 매월 관리자 1명이 2~3일 소요 | 매월 |
| **근로계약서 관리** | 워드 파일 템플릿에 이름·입사일만 변경. 프린트 후 서명·스캔·파일링 | 입사 시 |
| **안전교육 관리** | 교육 참석자 명단을 종이에 서명 → 사진 촬영 후 폴더 보관 | 분기별 |
| **위험성평가** | 고용노동부 제공 엑셀 양식에 위험요인 기입. 연 1회 작성 후 보관 | 연 1회 |
| **사고 보고** | 사고 발생 시 워드로 사고보고서 작성. 사진 첨부하여 이메일 보고 | 사고 시 |

**고객의 실제 이용 패턴 (시나리오):**
```
[30인 물류업 - 엑셀 관리 시나리오]

관리 인원: 관리부 직원 1명 (인사+총무+안전 겸직)

평상시:
- 출퇴근: 카카오톡으로 "출근합니다" → 관리직원이 엑셀에 수기 입력
- 초과근무: 현장 팀장이 구두로 "오늘 야근했다" 전달 → 관리직원 엑셀 정리
- 안전관리: "별일 없으면 안 함". 형식적 체크리스트 연 1~2회 작성
- 안전교육: 유튜브 영상 시청 + 참석 서명. 내용·이수증 관리 안 됨

사고 발생 시:
- 관리직원이 노무사에 전화 → 노무사가 "관련 자료 모아주세요" 요청
- "근태 기록 어디 있지?" → 엑셀 파일 여러 개 뒤짐 → 누락 발견
- "안전교육 기록 있나?" → 사진 1장만 남아 있음
- "이 직원의 근무 히스토리?" → 작년 엑셀 파일 삭제됨

→ 결과: 법적 대응 자료 준비에 2~3주 허비
→ 과태료 부과 가능성 급증
```

### 2.4.2 고객이 엑셀을 선택하는 이유

| 선택 이유 | 설명 | 강도 |
|-----------|------|------|
| **비용 제로** | "무료"라는 인식. Microsoft 365 라이선스 비용도 인지하지 않음 | 🔴 최강 |
| **학습 비용 제로** | 누구나 쓸 줄 안다. 새로운 것을 배울 필요 없음 | 🔴 매우 강함 |
| **유연성** | 자기 방식대로 커스터마이징 무한. 칸·수식·서식 자유 | 🟡 강함 |
| **"지금까지 문제 없었다"** | 사고 미발생 = 관리 잘하고 있다는 확증편향 | 🔴 매우 강함 |
| **관리자 1인 체제의 권력** | "엑셀 관리 = 나만 할 수 있는 일" → 시스템 도입 = 나의 가치 하락 우려 | 🟡 중간 |

### 2.4.3 엑셀의 한계 (SaaS 관점의 기회)

| 한계 | 구체적 상황 | SaaS 전환 가치 |
|------|------------|---------------|
| **데이터 유실** | 파일 삭제, PC 교체, 담당자 퇴사 시 데이터 소실 | 🔴 Critical — 법적 증빙 불가 |
| **버전 관리 불가** | "최종_최종_진짜최종.xlsx" — 어떤 것이 최신인지 불분명 | ✅ SaaS는 단일 진실의 원천(SSOT) |
| **동시 접근 불가** | 현장과 사무실이 동시에 수정 불가 | ✅ 클라우드 기반 실시간 동기화 |
| **자동화 한계** | 법 개정 시 수식 수동 변경 필요. 초과근무·야간수당 계산 오류 빈발 | ✅ 법 개정 자동 반영 |
| **리스크 사각지대** | 데이터가 연결되지 않으므로 "위험 패턴" 발견 불가 | ✅ 통합 데이터 기반 리스크 탐지 |
| **감사 부적합** | 노동청 근로감독·중대재해 조사 시 엑셀은 증거력 약함 | ✅ 시스템 로그 = 증거력 강화 |

### SaaS가 이길 수 있는 조건 — 대체재별 전환 트리거

| 대체재 | SaaS가 이기는 조건 | 전환이 일어나는 트리거 이벤트 | 전환 확률 |
|--------|-------------------|----------------------------|-----------|
| **엑셀** | ①이벤트 발생(사고·점검) ②관리자 퇴사로 데이터 유실 ③"위반사항 시각화"가 전환의 결정적 순간 | 노동청 점검 통보, 산재 발생, 관리담당 퇴사 | 🟡 30~40% (이벤트 의존) |
| **노무사** | ①SaaS가 노무사의 반복업무를 자동화하여 노무사 자체가 SaaS를 권할 때 ②비용 비교에서 SaaS가 월 비용 동등 이하일 때 ③노무사가 못하는 "사전 예방"을 증명할 때 | 노무사 비용 인상, 느린 대응에 불만, 2번째 사업장 개설 시 관리 복잡도 증가 | 🟡 20~30% (관계 전환 어려움) |
| **안전컨설팅** | ①정부 무상 컨설팅 종료 후 "사후관리" 니즈 발생 시 ②SaaS가 컨설팅의 이행관리 도구로 포지셔닝될 때 | 컨설팅 종료 → 노동청 점검 대비 불안, 정부 지원 미선정 | 🟢 40~50% (보완재 포지셔닝 가능) |
| **ERP** | ①ERP와 "경쟁"하지 않고 "병행" 가능할 때 ②ERP가 못하는 안전관리 영역만 공략할 때 ③API 연동으로 데이터 이중입력 제거 시 | ERP 리뉴얼 시점, 안전관리 기능 필요 인식 | 🟢 50~60% (보완재 포지셔닝 최적) |

### 대체재 전환 전략 매트릭스

```
                전환 난이도 (높음)
                     ▲
                     │
        노무사       │       ERP
        (관계 기반   │       (시스템 Lock-in
         전환 저항)  │        전환 비용 高)
                     │
    ─────────────────┼────────────────────▶ 보완 가능성 (높음)
                     │

```

# Part 3: 구매 의사결정 구조 분석

## 3.1 의사결정 구조 개요

한국 중소/중견기업(20~200인)의 HR + EHS SaaS 구매 의사결정은 **"대표이사 1인 결정 구조"**가 지배적이다. 대기업처럼 IT 부서, 구매팀, 법무팀이 참여하는 다단계 의사결정이 아니라, **"대표가 필요하다고 느끼면 산다"**는 단순한 구조다. 그러나 그 "필요하다고 느끼게 만드는 것"이 극도로 어렵다.

---

### 3.2.1 역할별 분석

| 역할 | 직위 (일반적) | 의사결정 영향력 | 제품에 대한 관심사 | 세일즈 접근 포인트 |
|------|-------------|---------------|-------------------|-------------------|
| **최종 결정자 (Decision Maker)** | 대표이사 / CEO | 🔴 **절대적** (90%+) | "이것 안 쓰면 어떤 일이 생기나?", "얼마나?", "당장 도입 가능한가?" | 리스크 시각화, 과태료·처벌 사례, 비용 대비 손실 방지 효과 |

| **외부 영향자 (External Influencer)** | 노무사 / 안전관리자 / 거래 세무사 | 🟡 **중간~높음** (신뢰 관계에 따라) | "내 업무에 도움이 되는가?", "나를 대체하는가?" | 전문가용 대시보드, "노무사가 10곳을 관리할 수 있게" |

## 3.4 가격 민감도 분석

### 3.4.1 가격 수용 구간

| 고객 세그먼트 | 월 수용 가격 (인당) | 월 수용 가격 (사업장 단위) | 가격 결정 기준 | 비교 대상 |
|-------------|-------------------|-------------------------|--------------|-----------|
| **20~50인 (제조)** | 3,000~5,000원 | 10~20만 원 | "노무사 비용보다 싸야" | 노무사 월 10~30만 원 |
| **50~100인 (제조)** | 5,000~10,000원 | 25~50만 원 | "인사 1명 월급보다 싸야" | 인사 담당 인건비 月 250~350만 원 |
| **100~200인 (건설/물류)** | 5,000~8,000원 | 50~100만 원 | "사고 한 건 비용보다 싸야" | 산재 1건 평균 비용 3,000만~1억 원 |

### 3.4.2 가격 심리 분석

| 심리 패턴 | 설명 | 대응 전략 |
|-----------|------|-----------|
| **앵커링 효과** | 고객은 첫 번째 접한 가격을 기준점으로 삼음. flex의 "인당 월 5,000원"이 시장 앵커 | 인당 과금이 아닌 **사업장 정액제**로 앵커를 피함 |
| **손실 회피 (Loss Aversion)** | "月 20만 원 절약"보다 "과태료 3,000만 원 방지"가 의사결정에 2.5배 강력 | "비용"이 아닌 **"보험"**으로 프레이밍 |
| **무료의 매직** | 무료 체험에서 유료 전환 시 심리적 장벽 급증 ("0원 → 10만 원"이 "10만 원 → 20만 원"보다 어려움) | 무료 진단(가치 확인) → **유료 체험**(1만 원이라도 과금) → 정규 과금으로 3단계 설계 |
| **비교 불가의 함정** | 통합 SaaS는 노무사·엑셀·EHS 어느 것과도 직접 비교 불가 → "비싸다/싸다" 판단 근거 부재 | **"비교 계산기"** 도구 제공: "현재 지출 (노무사+엑셀 관리시간+과태료 리스크) vs SaaS 비용" |

### 3.4.3 가격 모델 권장안

| 모델 | 구조 | 적합 세그먼트 | 장단점 |
|------|------|-------------|--------|
| **A. 사업장 정액제** | 月 15~30만 원 (50인 이하) / 30~50만 원 (100인 이하) | 제조업 | ✅ 예측 가능 비용 ✅ 인당 계산 불필요 ⚠️ 대규모 사업장에서 할인 요구 |
| **B. 인당 + 기본료** | 기본료 月 5만 원 + 인당 3,000~5,000원 | 물류/서비스업 | ✅ 규모 비례 공정 ⚠️ 인당 계산이 flex와 비교 유도 |
| **C. 가치 기반 (Premium)** | 月 50만 원+ (리스크 분석 리포트 + 전문가 연결 포함) | 100인+, 고위험 업종 | ✅ ARPU 高 ⚠️ 가치 증명까지 시간 필요 |

> **권장:** 초기에는 **A모델 (사업장 정액제)**로 시작. 이유: ①flex와의 인당 가격 비교를 피함 ②"월 15만 원"은 노무사 비용과 동등하면서 24/7 사용 가능 ③사장님이 인원 수 변동을 신경 쓸 필요 없음

## 3.5 도입 트리거 분석

### 3.5.1 트리거 유형별 분석

| 유형 | 트리거 이벤트 | 발생 빈도 | 전환 강도 | 세일즈 대응 |
|------|-------------|-----------|-----------|------------|
| **🔴 위기형 (Crisis)** | 산업재해 발생, 사망사고 발생 | 낮음 | 🔴 최강 — 즉각 구매 | 즉시 대응 가능한 "비상 패키지" 필요. 무료 긴급 진단 → 빠른 도입 |
| **🟠 규제형 (Regulatory)** | 노동청 근로감독 통보, 산재 보고서 제출 요구 | 중간 | 🔴 매우 강함 | "점검 대비 자료 자동 생성" 기능을 핵심 세일즈 포인트로 |
| **🟡 예방형 (Preventive)** | 법 개정 뉴스 시청, 동종업계 사고 뉴스, 노무사·세무사 조언 | 높음 | 🟡 중간 | 콘텐츠 마케팅(블로그·웹세미나)으로 "자각 → 진단 → 도입" 퍼널 구성 |
| **🟢 효율형 (Efficiency)** | 관리자 퇴사, 사업장 추가, 직원 수 증가, 엑셀 오류 발생 | 높음 | 🟡 중간 | "엑셀 탈출" 캠페인, 관리 업무 시간 절감 ROI 계산기 |
| **🔵 외부 압력형 (External)** | 원청사의 안전관리 시스템 도입 요구, ESG 평가 대비, 투자 심사 | 낮음 | 🟡 중간~강함 | 원청 대상 영업 → 하청 확산 모델 (네트워크 전략) |

### 3.5.2 트리거별 세일즈 시나리오

#### 시나리오 A: 위기형 트리거 → 최단 세일즈

```
DAY 0: 산재 발생 → 대표이사 패닉
DAY 1: 노무사에 연락 → 노무사가 "자료 정리가 안 되어 있다" 지적
DAY 2: 노무사가 "이런 시스템 있으면 편한데" 언급 (노무사가 채널인 경우)
DAY 3: 대표이사가 직접 검색 또는 관리부장에게 "알아봐라" 지시
DAY 5: 무료 진단 신청 → 위반 사항 시각화 리포트 수령
DAY 7: 대표이사 "바로 하자" → 도입 결정

세일즈 사이클: 1주일
핵심: "비상 시 즉각 대응 가능한 온보딩 속도"가 경쟁력
```

#### 시나리오 B: 예방형 트리거 → 표준 세일즈

```
WEEK 1: 중대재해처벌법 관련 뉴스/블로그 접촉 (콘텐츠 마케팅)
WEEK 2: "우리 회사는 괜찮나?" → 무료 리스크 진단 신청
WEEK 3: 진단 결과 수령 → "법적 위반 가능 항목 3건" 확인 → 관리부장에 전달
WEEK 4: 관리부장이 대표에 보고 + 무료 체험 시작
WEEK 6: 무료 체험 종료 → "계속 쓸까요?" → 대표 결정
WEEK 8: 유료 전환

세일즈 사이클: 2개월
핵심: "무료 진단"이 리드 생성의 핵심 도구
```

### 3.5.3 트리거 발생 캘린더 (연간)

| 월 | 주요 트리거 이벤트 | 마케팅/세일즈 액션 |
|----|-------------------|-------------------|
| **1~2월** | 중대재해처벌법 시행 기념일(1.27), 정부 무상 컨설팅 모집 시작 | 법 의무 콘텐츠 집중, "무료 진단" 캠페인 |
| **3~4월** | 상반기 근로감독 집중 시기, 건설업 성수기 시작 | "근로감독 대비" 타겟 광고, 건설 협력사 영업 |
| **5월** | 산업안전보건의 날(4.28), 안전교육 강화 기간 | 안전교육 관리 기능 프로모션 |
| **6~7월** | 폭염 특별 안전관리 기간, 하반기 인사이동 | 현장 안전 체크리스트 무료 제공, HR 기능 프로모션 |
| **9~10월** | 하반기 근로감독 집중 시기 | 2차 "근로감독 대비" 캠페인 |
| **11~12월** | 연말정산 시즌, 차년도 안전보건관리 계획 수립 | 급여/연말정산 자동화 + 차년도 안전계획 연계 프로모션 |

## 3.6 세일즈 전략 시사점

### 3.6.1 핵심 전략 원칙

| # | 원칙 | 구체적 실행 |
|---|------|------------|
| 1 | **"팔지 말고 진단하라"** | 첫 접점은 항상 "무료 리스크 진단". 세일즈 전화가 아니라 "위반 사항 리포트"가 첫 대화 | 
| 2 | **"관리부장이 아니라 대표를 움직여라"** | 세일즈 자료는 관리부장용(기능 비교표)과 대표용(리스크·비용 요약 1페이지)을 분리 |
| 3 | **"노무사를 적으로 만들지 마라"** | 노무사 제휴 프로그램: "노무사용 무료 계정 + 추천 수수료" → 노무사가 영업 채널 |
| 4 | **"현장을 이기는 제품이 이긴다"** | 관리부장이 좋아해도 현장 작업자가 안 쓰면 실패. 현장 시연이 세일즈 클로징의 핵심 |
| 5 | **"30분 안에 가치를 보여줘라"** | 가입 → 사업자번호 입력 → 해당 업종 법적 의무 자동 진단 → "당신의 위반 가능 항목" 출력까지 30분 이내 |

### 3.6.2 채널별 세일즈 전략

| 채널 | 전략 | 기대 효과 | 비용 효율 |
|------|------|-----------|-----------|
| **노무사 제휴** | 노무사에 무료 계정 + 추천 수수료(MRR의 10~20%). 노무사가 고객에 직접 추천 | CAC 최저, 신뢰 높음, 전환율 高 | 🟢 최우수 |
| **산업단지 네트워크** | 산업단지 관리공단, 중소기업 협동조합과 MOU → 단지 내 일괄 프로모션 | 집중된 타겟 접근, B2B 번들 가능 | 🟢 우수 |
| **콘텐츠 마케팅** | "중대재해처벌법 대응 가이드", "노동청 점검 체크리스트" 등 SEO 콘텐츠 | 리드 생성 자동화, 장기 자산 | 🟡 중간 |
| **정부 지원사업 연계** | 스마트공장 바우처, 비대면 서비스 바우처 대상 솔루션 등록 | 고객 비용 부담 제거, 대량 유입 | 🟡 중간 (행정 비용 高) |
| **원청 → 하청 확산** | 원도급 건설사·대형 물류사에 도입 → 협력사 의무 사용 | 네트워크 효과, Lock-in, 급속 확산 | 🟢 우수 (초기 세일즈 難) |
| **디지털 광고** | 구글/네이버 검색 광고 ("중대재해처벌법 대응", "HR SaaS" 등 키워드) | 즉시 리드 생성 | 🔴 높은 CPA |

## 최종 요약: 3개 보강 분석의 핵심 인사이트

| 분석 영역 | 핵심 결론 | 전략적 시사점 |
|-----------|----------|-------------|
| **시장 정의 검증** | 현재 정의는 적절. 환경(E), IoT HW, 채용을 명시 제외하면 TAM/SAM/SOM 확장 용이 | 3개 레이어(TAM/SAM/SOM)로 분해하여 사용할 것 |
| **대체재 심층** | 엑셀과 안전컨설팅은 "대체(Replace)" 전략, 노무사와 ERP는 "보완(Complement)" 전략으로 이원화 | 노무사를 채널로 전환하는 것이 CAC 최적화의 핵심 |
| **구매 의사결정** | 대표이사 1인 결정 구조. "위기형 트리거"가 가장 강력하나, "무료 진단"으로 예방형 트리거 생성 가능 | "팔지 말고 진단하라", 30분 내 첫 가치 경험 설계 필수 |

> **이 3개 분석은 기준 문서(1_Porter's 5 Forces Analysis)의 Force 3(대체재), Force 5(구매자 교섭력) 섹션의 전략적 깊이를 보강하며, 후속 TAM/SAM/SOM 분석 및 Go-To-Market 전략 수립의 입력 데이터로 활용될 수 있다.**
