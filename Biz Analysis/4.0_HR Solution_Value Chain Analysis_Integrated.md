# 시장 내 가치사슬 3-Layer 구조

현재 WRM 시장은 **3개 Layer가 분리**되어 있으며, 이를 연결하는 플레이어가 부재하다:

```
┌─────────────────────────────────────────────────────────────┐
│  ① Automation Layer (자동화 계층)                            │
│     HR SaaS — flex, 시프티                                   │
│     급여·근태·계약을 알고리즘으로 자동 처리                    │
├─────────────────────────────────────────────────────────────┤
│  ② Compliance Layer (규제 준수 계층)                         │
│     EHS SaaS — 세이프로, 스마플, 이지세이프                    │
│     안전·보건·환경 서류를 디지털로 기록·증빙                   │
├─────────────────────────────────────────────────────────────┤
│  ③ Human Layer (전문가 계층)                                 │
│     컨설팅 — 노무사, 안전관리 컨설턴트                        │
│     전문가의 지식·경험 기반 1:1 서비스                         │
├─────────────────────────────────────────────────────────────┤
│                        ❌ 단절                               │
│     현재 이 3개 Layer를 연결하는 플레이어가 없음               │
│     → 우리의 포지션 기회                                     │
└─────────────────────────────────────────────────────────────┘
```

# 경쟁사별 개별 가치사슬 분석

## 1) flex (플렉스)

### 1. 기업 개요

| 항목 | 내용 |
|------|------|
| **기업 유형** | B2B HR SaaS 스타트업 (시리즈 B) |
| **주요 고객** | 스타트업~중견기업 (IT/서비스업 중심, 20~500인) |
| **핵심 수익원** | 구성원 수 기반 월 구독료 (인당 5,000~15,000원) |

### 2. 가치사슬 분석

#### 2-1. 주요활동 (Primary Activities)

**① 투입/조달 물류 (Inbound Logistics)**

| 요소 | 내용 |
|------|------|
| **핵심 자산** | 한국 근로기준법·세법 도메인 지식, 급여 계산 로직, 4대보험 연동 API |
| **확보 방식** | 내재화 — 법률 전문가(노무사 출신) 내부 고용 + 법 개정 시 자체 업데이트 파이프라인 |
| **경쟁우위** | 법령 변경을 SaaS에 즉시 반영하는 속도. 6년간 축적된 한국 특화 규칙 엔진이 진입장벽 |

**② 운영/생산 (Operations)**

| 요소 | 내용 |
|------|------|
| **서비스 제공 방식** | 완전 자동화 SaaS. 고객이 직접 세팅·운영 (셀프서브 모델) |
| **핵심 프로세스** | 근태 데이터 수집 → 급여 자동 계산 → 전자계약 → 성과관리 → 비용관리. 모듈간 데이터 자동 연결 |
| **자동화 특징** | '시작 조건'과 '규칙'을 설정해 영수증 자동 지정, 자동 승인 등 업무 흐름을 스스로 처리하는 자동화 빌더 구조 |
| **비용 구조** | 고정비 중심 (개발 인건비 60%+, 클라우드 인프라 ~10%, 세일즈/마케팅 ~20%). 한계비용 극저 — 고객 1명 추가 비용 ≈ 0원 |

**③ 산출/배송 (Outbound Logistics)**

| 요소 | 내용 |
|------|------|
| **고객 전달 방식** | 100% 클라우드 웹+모바일 앱. 설치 불필요, 가입 즉시 사용 가능 |
| **사용자 경험** | 관리자(HR 담당) 중심 UX. 대시보드·리포트·알림 자동화. 현장 작업자 대상 UX는 부차적 |

**④ 마케팅 및 영업 (Marketing & Sales)**

| 요소 | 내용 |
|------|------|
| **주요 획득 채널** | 콘텐츠 마케팅(블로그·웹세미나), SEO("주 52시간" 등 키워드), 구전(스타트업 커뮤니티) |
| **세일즈 방식** | 셀프서브 + 인바운드 세일즈 혼합. 50인 이상은 영업팀 터치 |
| **추가 전략** | 무료 체험(Free Trial) 유도와 유명 기업 경영진의 성공 사례(Social Proof)를 활용한 인바운드 리드 획득 |
| **CAC 구조** | 콘텐츠 기반 인바운드 → CAC 상대적 低. 추정 CAC 30~50만 원/사. LTV:CAC 비율 우수 |

**⑤ 서비스 (Service)**

| 요소 | 내용 |
|------|------|
| **고객 지원** | 채팅 기반 CS, 헬프센터, 웹세미나 교육. 대규모 고객은 전담 CSM(Customer Success Manager) 배정 |
| **유지/업셀 전략** | Land & Expand — 근태로 진입 → 급여 → 전자계약 → 성과관리 → 비용관리 순차 업셀. 모듈 추가될수록 ARPU 증가 + 전환비용 상승 |

#### 2-2. 지원활동 (Support Activities)

| 활동 | 내용 |
|------|------|
| **기업 인프라** | 스타트업 조직 (CEO 직접 제품 리딩). 빠른 의사결정. 시리즈 B 투자자 이사회 참여 |
| **인적 자원** | 개발 인력이 전체의 50%+. PO(Product Owner) 중심 스쿼드 조직. 노무사 출신 도메인 전문가 내부 보유 |
| **기술 개발** | 풀스택 자체 개발. AI(법률 자문, 데이터 분석) 도입 추진. 법령 규칙 엔진 자체 구축 |
| **조달/구매** | AWS/GCP 클라우드 인프라 (매출 대비 5~10%). 외부 의존도 低 (핵심 기술 모두 내재화) |

### 3. 핵심 수익창출 포인트

> **"법령에 종속된 반복 업무의 자동화"가 돈을 버는 지점이다.**

flex가 실제로 가치를 창출하는 활동은 **"급여 계산의 정확성"**이다. 한국 노동법·세법의 복잡한 규칙(주 52시간, 포괄임금제, 야간/휴일 수당 계산, 4대보험 요율 변동)을 코드로 내재화한 규칙 엔진이 핵심 자산이다. 인사 담당자가 엑셀로 밤새 계산해야 할 복잡한 연장·야간·휴일 수당을 **단 한 번의 클릭으로 연산**해 내는 '정산 자동화 로직'에서 수익이 발생한다. 기업의 '행정 비용'을 '구독료'로 치환시키는 것이다.

고객이 한 번 급여를 flex로 처리하면, 과거 데이터 축적 + 재학습 비용으로 인해 전환이 극히 어려워진다.

### 4. 경쟁우위 및 약점

| 강점 | 약점 |
|------|------|
| 한국 법령 급여 규칙 엔진 6년 축적 | EHS/안전관리 역량 제로 |
| Land & Expand로 ARPU 지속 성장 구조 | 제조/건설/물류 현장 고객 부재 |
| 206명 개발팀의 제품 고도화 속도 | "현장 작업자"가 아닌 "사무직 관리자" 중심 UX |
| 5,000억 기업가치 → 자본력 확보 | 인당 과금 모델 → 중소기업 가격 민감도에 취약 |
| HR 행정 완전 자동화로 고객 인건비 절감 | 중대재해처벌법 대응 등 블루칼라 현장 리스크 방어 불가 |

### 5. 우리 사업에 대한 시사점

| 구분 | 내용 |
|------|------|
| **배울 점** | ① 법령 규칙 엔진의 내재화 전략 — 외부 노무사 의존 없이 규칙을 코드로 구현 ② Land & Expand 모듈 확장 전략 ③ 셀프서브 온보딩으로 CAC 최소화 ④ 자동화 빌더 구조 설계 |
| **피해야 할 전략** | "올인원 HR SaaS"로 포지셔닝하면 flex와 정면 충돌. 기능 수에서 절대 승리 불가 |
| **공격 가능 영역** | flex가 구조적으로 진입하기 어려운 "현장(제조/건설/물류)" + "안전관리" + "법적 리스크 통합". flex의 고객은 IT/서비스업 → 제조/건설은 아직 미개발 |

## 2) 시프티 (Shiftee)

### 1. 기업 개요

| 항목 | 내용 |
|------|------|
| **기업 유형** | B2B 초경량·자동화 집중형 HR SaaS |
| **주요 고객** | 30인 미만 영세기업부터 교대근무 복잡한 대기업(SK, 롯데, 카카오)까지 전천후 |
| **핵심 수익원** | 사용자 수 기반 월 구독료 (인원 비례 소프트웨어 라이선스) |

### 2. 가치사슬 분석

#### 2-1. 주요활동

| 활동 | 내용 |
|------|------|
| **투입/조달** | **핵심 자산:** 근태 정밀 계산 알고리즘 (교대·유연·탄력 근무 대응), 위치/WiFi 인증 기술. 물리적 장비 없이 스마트폰 GPS, 사내 WiFi, SECOM 등 API를 연동해 위치/시간 인증 데이터 조달. **확보:** 100% 자체 개발. **경쟁우위:** 복잡한 교대 패턴 처리 정밀도 업계 최고 수준 |
| **운영/생산** | **방식:** 완전 자동화 SaaS. 장비 없이 스마트폰만으로 출퇴근 인증. **100여 개 이상의 수당 집계 옵션**을 고객이 직접 설정하게 하여, 수동 개입(SI) 없이 모든 급여 연산이 알고리즘으로 자동 생산. **비용:** 극도로 린(Lean)한 구조 — 11억 투자로 흑자 달성. **영업이익률 74.5%**로 SaaS 업계 최상위 수익성. 커스터마이징을 **철저히 배제**하는 단호한 의사결정 구조 |
| **산출/배송** | 웹+모바일 앱 + PC-OFF 앱. 별도 하드웨어 불필요 (위치/WiFi 기반). 디지털 리포트 형태로 즉각 배송 |
| **마케팅/영업** | PLG(제품 주도 성장), 구전, SEO. 최소 영업 인력. 재구매율 97.2%로 자연 유지. **30인 미만 무료**, **정부 클라우드 바우처** 적극 활용 → 도입 비용 장벽 소거. 인바운드 리드 폭발적 획득 |
| **서비스** | 법령 변화에 맞춘 자동 업데이트 + 슬랙 등 메신저 연동으로 인적 CS 최소화. 대기업 고객은 전담 지원. 기능 단순→교육 비용 低 |

#### 2-2. 지원활동

| 활동 | 내용 |
|------|------|
| **기업 인프라** | 소규모 린 조직 (~50명). 비즈니스온 인수 → 재독립 과정에서 자생력 입증 |
| **인적 자원** | 개발 중심. 영업/마케팅 인력 최소. "적은 인원으로 높은 수익" 모델. 50여 명으로 수십만 고객사 통제 |
| **기술 개발** | 근태 스케줄링 알고리즘이 핵심 IP. 글로벌(대만) 다국적 근무 패턴 대응. 저사양 기기 최적화 R&D |
| **조달/구매** | 클라우드 인프라 비용 극소. 외부 의존도 최저 수준 |

### 3. 핵심 수익창출 포인트

> **"복잡한 근태 패턴의 정밀 처리"가 돈을 버는 지점이다.**

시프티는 "근태 한 가지"를 극도로 잘한다. 3교대, 유연근무, 탄력근무, 선택근무 등 한국 근로기준법의 복잡한 근무 유형을 정확히 계산하고, 장비 없이 스마트폰만으로 출퇴근을 인증한다. 이 "단일 기능의 깊이"가 97.2% 재구매율의 원천이다. 매출이 증가해도 비용이 늘지 않아 **74.5%라는 경이로운 영업이익률**을 달성한다.

### 4. 경쟁우위 및 약점

| 강점 | 약점 |
|------|------|
| 영업이익률 74.5% — SaaS 업계 최상위 수익성 | 근태 외 기능 확장이 느림 |
| 장비 없는 출퇴근 인증 → 도입 장벽 제로 | EHS/안전관리 역량 전무 |
| 대기업(SK, 롯데) 레퍼런스 확보 | 제조/건설 현장 특화 UX 부재 |
| 97.2% 재구매율 → 자연 유지 수익 | 근태 한 가지에 과도 집중 → 성장 한계 |
| 궁극의 자동화와 표준화 | 산업 현장의 전사적 리스크 관리 불가 |

### 5. 우리 사업에 대한 시사점

| 구분 | 내용 |
|------|------|
| **배울 점** | ① 린(Lean) 경영 — 11억 투자로 흑자 달성한 비용 구조 ② "잘 하는 것 하나"에 집중하는 버티컬 전략 ③ 장비 불필요 모바일 인증의 도입 용이성 ④ 커스터마이징 **철저 거부** 원칙 ⑤ 30인 미만 무료 + 정부 바우처 활용 전략 |

안전하이는 "안전관리자가 24시간 현장을 볼 수 없다"는 물리적 한계를 AI로 해결한다. 관리자의 육안을 대체하는 24시간 실시간 감시로 중대재해를 선제 차단한다. 그러나 중소기업에게는 **"우리 현장에 CCTV가 제대로 없다"**가 도입 장벽이 된다.

### 4. 경쟁우위 및 약점

| 강점 | 약점 |
|------|------|
| 기존 CCTV 활용 → 추가 장비 불필요 | 막대한 도입 단가·유지보수 비용 → 중소기업 진입 불가 |
| 14개 언어 AI 번역 → 외국인 근로자 Pain 해결 | HR 기능 전무. 급여·근태 관리 불가 |
| 데이터 융합(CCTV+날씨+작업지시) → 사고 예측 | 기업 규모·재무 정보 불투명. 사업 지속성 리스크 |
| 생성AI 활용 → 위험성평가 자동 생성 | GPU 비용·현장 커스터마이징 비용 높은 비용 구조. 이익률 낮음 |
| 24시간 실시간 감시 기능 | 하드웨어 종속성 |

### 5. 우리 사업에 대한 시사점

| 구분 | 내용 |
|------|------|
| **배울 점** | ① AI 기반 위험성평가 자동 생성 — 향후 프리미엄 기능 후보 ② 외국인 근로자 다국어 지원 — 제조/건설의 실질적 Pain |
| **피해야 할 전략** | AI 기반 영상 분석을 MVP에 포함하면 비용 폭증 + 기술 리스크 증가. CCTV 연동은 Phase 3 이후 |
| **공격 가능 영역** | 안전하이가 못하는 "일상적 안전관리(교육·점검·서류)" + "HR 통합". 안전하이는 "감시" 중심, 우리는 "관리" 중심으로 차별화 |

## 6) 이지세이프 (EASY SAFE)

### 1. 기업 개요

| 항목 | 내용 |
|------|------|
| **기업 유형** | 네트워크 중심 현장 안전관리 모바일 SaaS (초기 스타트업) |
| **주요 고객** | 건설업/제조업 내 원청 및 하청(수급인) 중소규모 현장 |
| **핵심 수익원** | 사용 인원수 비례 월 구독료 |

### 2. 가치사슬 분석

#### 2-1. 주요활동

| 활동 | 내용 |
|------|------|
| **투입/조달** | **핵심 자산:** '원청과 하청업체 간의 연대 책임 의무'라는 중대재해처벌법 법적 로직을 핵심 자산으로 시스템 내재화. 원/하청 안전관리 프로세스 지식. **경쟁우위:** "원청의 법적 책임 방어"라는 명확한 Pain 해결 |
| **운영/생산** | 모바일 앱 중심 SaaS. 복잡성을 덜어내고 **고령자·외국인도 즉시 사용 가능하도록 UI/UX를 극단적으로 단순화**. 위험성평가, 안전점검, 근로자 컨디션 체크를 모바일로 수행. 소규모 개발팀 린 운영 |
| **산출/배송** | 철저히 근로자 스마트폰(모바일 앱) 중심. 실시간 점검 데이터가 클라우드를 거쳐 원청 관리자에게 자동 배송. 시니어·외국인 대응 직관적 UI |
| **마케팅/영업** | **원청업체 1곳을 뚫으면 산하 하청업체가 의무적으로 가입하는 '구조적 바이럴 루프(Viral Loop)'** → 고객 획득 비용 극도로 낮춤. 행정 서류 업무 85% 절감 가치 제안 |
| **서비스** | 앱 단순성 덕분에 CS 최소화. 기능 유지 중심 서비스 |

#### 2-2. 지원활동

| 활동 | 내용 |
|------|------|
| **기업 인프라** | 소규모 스타트업 조직. 모바일 프론트엔드 개발자 위주의 가벼운 인프라 |
| **인적 자원** | 소규모 개발팀 + 안전관리 도메인 지식 보유자 (추정) |
| **기술 개발** | 모바일 UX에 집중. 태그 공유, 외국인/시니어 대응 UI가 기술적 차별점. 저사양 기기 최적화 |
| **조달/구매** | 클라우드 인프라 비용 소규모. 외부 의존도 低 |

### 3. 핵심 수익창출 포인트

> **"원청 도급사의 법적 책임 방어 증빙 데이터"와 "원-하청 네트워크 효과"가 돈을 버는 지점이다.**

이지세이프의 실질적 가치는 두 가지다:
1. "원청이 하청 작업자의 안전관리를 했다"는 **증거를 디지털로 남기는 것** (법적 면책 증빙)
2. 원청 1곳 가입 → 하청업체 **의무적 가입** → **구조적 바이럴 루프 → 영업비용 제로화**

원-하청 생태계를 묶어버리는 네트워크 효과로 행정 서류 업무를 **85% 절감**시키는 점이 강력한 무기다.

### 4. 경쟁우위 및 약점

| 강점 | 약점 |
|------|------|
| 원/하청 통합 관리라는 명확한 니치 | HR 기능 전무 → 확장성 제한 |
| 시니어·외국인 근로자 대응 UX | 기업 정보·재무 불투명 → 사업 지속성 리스크 |
| "EASY + SAFE" 양면 가치 제안 | 시장 인지도 극히 낮음 |
| 원-하청 바이럴 루프 → 영업비용 극소 | 특정 현장 공사 기간에만 사용되는 휘발성 솔루션 전락 위험 |
| 행정 서류 85% 절감 | HR 코어(ERP) 기능 없음 |

### 5. 우리 사업에 대한 시사점

| 구분 | 내용 |
|------|------|
| **배울 점** | ① 원/하청 "도급사 법적 방어"라는 Pain의 명확한 정의 ② 시니어·외국인 대응 UX 설계 원칙 ③ **원-하청 바이럴 루프** 시스템 설계 → 우리 제품에도 원/하청 구조 내재화 필요 |
| **피해야 할 전략** | 원/하청 구조에만 특화하면 시장이 지나치게 좁아짐. 니치에서 시작하되 확장 경로 필수 |
| **공격 가능 영역** | 이지세이프 고객이 "근태·급여도 관리하고 싶다"는 확장 니즈 발생 시 대체 가능. 공사 종료 후 휘발되는 약점을 '상시 HR 데이터'로 보완 |

# 시장 내 가치사슬 구조 유형 3가지 정의

## 유형 A: "소프트웨어 자동화형" — 고수익형 (Software Automation Model)

> **대표 기업:** flex, 시프티

```
[법령/도메인 지식] → [규칙 엔진 코드화] → [SaaS 자동 처리] → [고객 셀프서브]
     (투입)              (운영)              (산출)           (서비스)
```

| 특성 | 설명 |
|------|------|
| **가치 원천** | 복잡한 법률/규칙을 소프트웨어로 자동화. "사람이 할 일을 소프트웨어가 대신" |
| **생산 방식** | 하드웨어와 인적 개입을 완전 배제. API 연동과 코드 베이스에 집중. 셀프서브 강제로 커스터마이징 100% 차단 |
| **비용 구조** | 고정비 중심 (개발 인건비). 한계비용 ≈ 0원. 규모의 경제 극대화 |
| **확장 방식** | 기능 모듈 추가 (Land & Expand). 수평적 확장. 글로벌 무한 확장 가능 |
| **약점** | "현장"을 모름. 모든 가치가 사무실 안에서 완결됨. 물리적 현장 접점 제로 |
| **수익 구조** | 인당/사업장 구독료. MRR/ARR 기반. **영업이익률 70% 이상** (시프티 74.5% 실증) |

| **영업이익률** | **70%+** | 20~40% | 낮음 |
| **데이터 연결** | HR 데이터만 | EHS 데이터만 | 영상/센서 데이터만 |

# 기업 유형별 가치사슬 비교 분석

## (1) 유형별 비용 구조 비교

| 비용 항목 | HR SaaS | ERP | EHS SaaS | 컨설팅 |
|-----------|:-------:|:---:|:--------:|:------:|
| **개발/R&D** | 매출 대비 40~50% | 30~40% | 35~45% | 0% (개인역량) |
| **인프라** | 5~10% | 3~5% (On-Prem) | 5~15% (하이브리드) | 0% |
| **영업/마케팅** | 15~25% | 20~30% | 20~35% | ~5% (관계 기반) |
| **고객지원/CS** | 5~10% | 10~15% | 10~20% | 0% (자문 = 운영) |
| **고정비 : 변동비** | **85:15** | **70:30** | **60:40** | **10:90** |
| **한계비용** | **≈ 0원** | 중간 (구축 인력) | 낮음~중간 | **= 인건비** |

## (2) 확장성 비교

| 확장 차원 | HR SaaS | ERP | EHS SaaS | 컨설팅 |
|-----------|:-------:|:---:|:--------:|:------:|
| **고객 확장 속도** | 🟢 매우 빠름 (셀프서브) | 🔴 느림 (구축 필요) | 🟡 중간 (현장 온보딩) | 🔴 극히 느림 (1:1) |
| **기능 확장** | 🟢 모듈 추가 용이 | 🟡 가능하나 무거움 | 🟡 별도 제품화 경향 | 🔴 불가 (사람=한계) |
| **지역 확장** | 🟢 클라우드 = 즉시 | 🟡 구축 인력 필요 | 🟡 현장 지원 필요 | 🔴 물리 이동 한계 |
| **매출 천장** | 🟢 없음 (규모의 경제) | 🟡 조직 성장 종속 | 🟡 SaaS=무한, 구축=유한 | 🔴 인력 수 = 천장 |

## (3) 수익성 비교

| 수익성 지표 | HR SaaS | ERP | EHS SaaS | 컨설팅 |
|------------|:-------:|:---:|:--------:|:------:|
| **매출총이익률** | **75~85%** | 50~65% | 55~70% | 60~80% |
| **영업이익률** | **20~40%** (시프티 74.5%) | 25~30% (더존 28.6%) | 10~25% (추정) | 30~50% (개인) |
| **수익 반복성** | 🟢 MRR/ARR 95%+ | 🟡 유지보수 10~20% | 🟡 SaaS=반복, 구축=일시 | 🔴 고문=반복, 사건=일시 |
| **LTV:CAC** | 🟢 5:1 이상 | 🟡 3:1 추정 | 🟡 2~4:1 추정 | 🟢 매우 높음 (CAC≈0) |

## (4) 가장 효율적인 가치사슬 구조

### 결론: **HR SaaS형 (유형 A — 소프트웨어 자동화형)이 가장 효율적이다**

| # | 효율성 원리 | HR SaaS가 우월한 이유 |
|---|------------|---------------------|
| 1 | **한계비용 제로** | 고객 1명 추가 시 비용 ≈ 0원 |
| 2 | **고정비/변동비 85:15** | 매출 성장 시 비용 정체 → 영업 레버리지 극대화 |
| 3 | **셀프서브 온보딩** | 고객이 직접 가입·세팅 → CAC 최저 |
| 4 | **Land & Expand** | 저가 모듈 진입 → ARPU 2~5배 성장 |
| 5 | **반복 수익 95%+** | MRR/ARR 기반 자동 과금 → 매출 예측 가능 |

> **증거: 시프티** — 11억 투자 → 매출 66억 → 영업이익 49.6억 → 영업이익률 74.5% → 직원 1인당 영업이익 1억 원

# 핵심 수익 활동 추출 및 "가장 돈 되는 구조" 정의

## (1) 경쟁사별 수익 발생 지점

| 기업 | 수익 발생 지점 | 고객이 돈을 내는 진짜 이유 | 수익 유형 |
|------|:---:|:---:|:---:|
| **flex** | 운영 → 급여 자동 계산 | "급여를 틀리면 소송 → flex가 정확히 계산" | 🟢 반복 구독 |
| **시프티** | 운영 → 근태 패턴 처리 | "3교대 스케줄링이 복잡 → 시프티만 정확히 처리" | 🟢 반복 구독 |
| **세이프로** | 산출 → 현장 모바일 기록 | "안전 서류가 아직 종이 → 모바일로 바로 작성" | 🟡 구독+구축 혼합 |
| **스마플** | 투입 → 협회 표준 프로세스 | "협회가 만들었으니 법적으로 안전" | 🟡 구독 (신뢰 프리미엄) |
| **안전하이** | 운영 → AI 영상 분석 | "사람의 눈이 닿지 않는 곳을 AI가 감시" | 🔴 프로젝트+라이선스 |
| **이지세이프** | 산출 → 도급사 증빙 + 바이럴 | "원청 법적 방어 데이터 + 하청 자동 편입" | 🟢 구독 (네트워크 효과) |

### 수익 발생의 3가지 핵심 메커니즘

1. **인건비와 시간의 알고리즘화 (HR SaaS)**: 인사 담당자가 밤새 엑셀로 계산할 복잡한 수당을 한 클릭으로 연산. 기업의 '행정 비용'을 '구독료'로 치환
2. **규제 공포의 문서적 방어 (EHS SaaS/컨설팅)**: 사고 시 경영진 구속 공포를 해결. TBM, 위험성 평가 등 법정 서류를 빈틈없이 생성·보관 → **'법적 면책의 보험'**을 판매
3. **네트워크 강제 편입 (이지세이프 유형)**: 원청 지위를 이용해 하청업체에 플랫폼 가입을 강제하는 **'구조적 락인'** 자체가 영업비용을 없애고 유료 시트를 발생

## (2) 스케일 후 이익률 예측 (고객 1,000곳+)

| 구조 | 현재 | 1,000곳 도달 시 | 이유 |
|------|:----:|:--------------:|------|
| **HR SaaS** | 20~40% | **50~70%** | 한계비용 0 → 매출 증가 = 이익 증가 |
| **EHS SaaS (SaaS Only)** | 10~25% | **35~50%** | 구축형 폐지 시 HR SaaS에 근접 가능 |
| **ERP** | 25~30% | **30~35%** | 구축 인력 비례 성장 → 이익률 상한 존재 |
| **컨설팅** | 30~50% | ❌ 도달 불가 | 인력 = 천장. 물리적으로 1,000곳 서비스 불가 |

## (3) 최종: "가장 돈 되는 구조" 정의

```
┌───────────────────────────────────────────────────────────────────┐
│                                                                    │
│              ★ "가장 돈 되는 구조" 공식 ★                        │
│                                                                    │
│   수익 = (법령 규칙 엔진의 깊이) × (자동화율) × (데이터 연결 가치) │
│          ─────────────────────────────────────────────────────────  │
│                             인력 투입 비용                          │
│                                                                    │
│   구체적으로:                                                      │
│   ① 법령 규칙을 코드로 내재화할수록 → 진입장벽 ↑ + 전환비용 ↑     │
│   ② 자동화율이 높을수록 → 한계비용 ↓ + 이익률 ↑                   │
│   ③ 데이터가 연결될수록 → 차별화 ↑ + ARPU ↑                       │
│   ④ 인력 투입이 적을수록 → 비용 ↓ + 확장 속도 ↑                   │
│                                                                    │
│   이 네 변수를 동시에 최적화하는 유일한 구조:                      │
│                                                                    │
│   「SaaS Only + 법령 규칙 엔진 + HR↔EHS 데이터 연결               │
│     + 전문가 마켓플레이스 중개 (원가 외주화)」                     │
│                                                                    │
└───────────────────────────────────────────────────────────────────┘
```

# 우리가 들어갈 수 있는 가치사슬 포지션

## 제안: "유형 A + 유형 B의 연결자" (Bridge Model)

현재 시장의 구조적 문제는 **유형 A(사무실 HR)와 유형 B(현장 EHS)가 완전히 단절**되어 있다는 것이다. 어떤 기업도 이 두 유형을 연결하지 않고 있다.

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                  │
│   유형 A의 가치                    유형 B의 가치                │
│   (사무실 HR)                      (현장 EHS)                   │
│                                                                  │
│   급여 계산 ─────┐          ┌───── 위험성평가                    │
│   근태 관리 ─────┤          ├───── 안전교육                      │
│   근로계약 ─────┤          ├───── 사고기록                      │
│   연차 관리 ─────┤          ├───── 법정점검                      │
│                   │          │                                    │
│                   │   ???    │                                    │
│                   │          │                                    │
│                   └──── ❌ ──┘                                    │
│                     단절 지점                                     │
│                   (아무도 연결하지 않음)                          │
│                                                                  │
│                        ▼ ▼ ▼                                     │
│                                                                  │
│                   ★ 우리의 포지션 ★                             │
│                                                                  │
│                   [리스크 통합 분석 레이어]                       │
│                                                                  │
│   급여 데이터 ────┐          ┌───── 안전 데이터                  │
│   근태 데이터 ────┼── 연결 ──┤                                   │
│   계약 데이터 ────┤     ↕    ├───── 사고 데이터                  │
│                   │  리스크  │                                    │
│                   │ 프로파일 │                                    │
│                   └─────────┘                                    │
│                        │                                         │
│                        ▼                                         │
│               [법적 대응 리포트]                                 │
│               [리스크 예측 알림]                                  │
│               [전문가 연결 생태계]                                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 구체적 포지션 정의

| 차원 | 우리의 포지션 |
|------|-------------|
| **투입/조달** | 노동법(유형 A 자산) + 산업안전보건법(유형 B 자산) **모두** 내재화. 100% 클라우드 및 API 연동 |
| **운영/생산** | "충분히 좋은" HR(급여·근태) + "충분히 좋은" EHS(교육·점검·사고) + **"유일한" 리스크 통합 분석**. 노코드 범용 설정만 업데이트. **커스텀/구축형 절대 거절** |
| **산출/배송** | 관리자용 PC 대시보드 + 현장용 모바일 앱 **동시 제공**. 데이터 실시간 연결 |
| **마케팅/영업** | 노무사·안전관리자 = 채널. "무료 리스크 진단" = 리드. 정부 바우처 활용 + **원/하청 바이럴 루프 내재화** |
| **서비스** | 전문가 마켓플레이스(노무사·안전관리자가 제품 위에서 서비스). 중개 수수료만 취하고 원가 전가 |

### 핵심 차별화: "데이터 연결의 가치"

경쟁사 누구도 제공하지 않는 가치:

| 데이터 연결 시나리오 | 가치 |
|---------------------|------|
| 근태 이상(잦은 결근) + 현장 아차사고 이력 → **고위험 직원 식별** | 사고 사전 예방 |
| 초과근무 데이터 + 사고 발생 시간대 → **피로 기반 리스크 예측** | 작업 배치 최적화 |
| 안전교육 미이수 + 위험 작업 배정 → **법적 위반 자동 경고** | 과태료 사전 방지 |
| 근로계약 만료 + 안전인증 만료 → **행정 기한 통합 관리** | 관리 업무 70% 감소 |
| 전체 데이터 → **노동청 점검·산재 대응 리포트 자동 생성** | 법적 대응 비용 절감 |

## 종합: 양측 모두 공략 가능

```
[공략 우선순위]

1순위: 이지세이프 유형 (전환 비용 最低 → 흡수 용이)
     → "안전관리 앱은 쓰는데, 근태·급여는 여전히 엑셀"
     → 우리: "안전관리 + 근태 + 급여를 하나에서. 데이터도 연결"

2순위: 스마플 유형 (HR 분절 불만 → 교체 동기 존재)
     → "안전 서류만 보관하는 비용으로 인사까지 원스톱"
     → 무거운 컨소시엄을 가벼운 통합 SaaS로 Rip & Replace

공격하지 않는 유형: flex (방어력 最強 — 급여 Lock-in + 자본력)
```

### 가장 방어력이 강한 유형 (flex)

| 방어 요인 | 설명 |
|-----------|------|
| 급여 데이터 축적 → 전환 비용 극高 | 3년치 급여 이력은 타 시스템으로 이전 극난 |
| Land & Expand로 모듈 깊이 증가 → Lock-in 강화 | 모듈 5개 이상 사용 고객의 이탈률 ≈ 0% |
| 206명 개발 조직의 속도 | 경쟁사가 추격 어려운 기능 확장 속도 |
| 기업가치 5,000억의 자본 방어벽 | 가격 전쟁에서도 버틸 여력 |
