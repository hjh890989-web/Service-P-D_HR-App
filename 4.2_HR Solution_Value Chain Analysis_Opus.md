# 경쟁사별 개별 가치사슬 분석

## 1) flex (플렉스)

### 1. 기업 개요

| 항목 | 내용 |
|------|------|
| **기업 유형** | B2B HR SaaS 스타트업 (시리즈 B) |
| **주요 고객** | 스타트업~중견기업 (IT/서비스업 중심, 20~500인) |
| **핵심 수익원** | 구성원 수 기반 월 구독료 (인당 5,000~15,000원) |

### 2. 가치사슬 분석

#### 2-1. 주요활동 (Primary Activities)

**① 투입/조달 물류 (Inbound Logistics)**

| 요소 | 내용 |
|------|------|
| **핵심 자산** | 한국 근로기준법·세법 도메인 지식, 급여 계산 로직, 4대보험 연동 API |
| **확보 방식** | 내재화 — 법률 전문가(노무사 출신) 내부 고용 + 법 개정 시 자체 업데이트 파이프라인 |
| **경쟁우위** | 법령 변경을 SaaS에 즉시 반영하는 속도. 6년간 축적된 한국 특화 규칙 엔진이 진입장벽 |

**② 운영/생산 (Operations)**

| 요소 | 내용 |
|------|------|
| **서비스 제공 방식** | 완전 자동화 SaaS. 고객이 직접 세팅·운영 (셀프서브 모델) |
| **핵심 프로세스** | 근태 데이터 수집 → 급여 자동 계산 → 전자계약 → 성과관리 → 비용관리. 모듈간 데이터 자동 연결 |
| **비용 구조** | 고정비 중심 (개발 인건비 60%+, 클라우드 인프라 ~10%, 세일즈/마케팅 ~20%). 한계비용 극저 — 고객 1명 추가 비용 ≈ 0원 |

**③ 산출/배송 (Outbound Logistics)**

| 요소 | 내용 |
|------|------|
| **고객 전달 방식** | 100% 클라우드 웹+모바일 앱. 설치 불필요, 가입 즉시 사용 가능 |
| **사용자 경험** | 관리자(HR 담당) 중심 UX. 대시보드·리포트·알림 자동화. 현장 작업자 대상 UX는 부차적 |

**④ 마케팅 및 영업 (Marketing & Sales)**

| 요소 | 내용 |
|------|------|
| **주요 획득 채널** | 콘텐츠 마케팅(블로그·웹세미나), SEO("주 52시간" 등 키워드), 구전(스타트업 커뮤니티) |
| **세일즈 방식** | 셀프서브 + 인바운드 세일즈 혼합. 50인 이상은 영업팀 터치 |
| **CAC 구조** | 콘텐츠 기반 인바운드 → CAC 상대적 低. 추정 CAC 30~50만 원/사. LTV:CAC 비율 우수 |

**⑤ 서비스 (Service)**

| 요소 | 내용 |
|------|------|
| **고객 지원** | 채팅 기반 CS, 헬프센터, 웹세미나 교육. 대규모 고객은 전담 CSM(Customer Success Manager) 배정 |
| **유지/업셀 전략** | Land & Expand — 근태로 진입 → 급여 → 전자계약 → 성과관리 → 비용관리 순차 업셀. 모듈 추가될수록 ARPU 증가 + 전환비용 상승 |

#### 2-2. 지원활동 (Support Activities)

| 활동 | 내용 |
|------|------|
| **기업 인프라** | 스타트업 조직 (CEO 직접 제품 리딩). 빠른 의사결정. 시리즈 B 투자자 이사회 참여 |
| **인적 자원** | 개발 인력이 전체의 50%+. PO(Product Owner) 중심 스쿼드 조직. 노무사 출신 도메인 전문가 내부 보유 |
| **기술 개발** | 풀스택 자체 개발. AI(법률 자문, 데이터 분석) 도입 추진. 법령 규칙 엔진 자체 구축 |
| **조달/구매** | AWS/GCP 클라우드 인프라 (매출 대비 5~10%). 외부 의존도 低 (핵심 기술 모두 내재화) |

### 3. 핵심 수익창출 포인트

> **"법령에 종속된 반복 업무의 자동화"가 돈을 버는 지점이다.**

flex가 실제로 가치를 창출하는 활동은 **"급여 계산의 정확성"**이다. 한국 노동법·세법의 복잡한 규칙(주 52시간, 포괄임금제, 야간/휴일 수당 계산, 4대보험 요율 변동)을 코드로 내재화한 규칙 엔진이 핵심 자산이다. 고객이 한 번 급여를 flex로 처리하면, 과거 데이터 축적 + 재학습 비용으로 인해 전환이 극히 어려워진다.

### 4. 경쟁우위 및 약점

| 강점 | 약점 |
|------|------|
| 한국 법령 급여 규칙 엔진 6년 축적 | EHS/안전관리 역량 제로 |
| Land & Expand로 ARPU 지속 성장 구조 | 제조/건설/물류 현장 고객 부재 |
| 206명 개발팀의 제품 고도화 속도 | "현장 작업자"가 아닌 "사무직 관리자" 중심 UX |
| 5,000억 기업가치 → 자본력 확보 | 인당 과금 모델 → 중소기업 가격 민감도에 취약 |

### 5. 우리 사업에 대한 시사점

| 구분 | 내용 |
|------|------|
| **배울 점** | ① 법령 규칙 엔진의 내재화 전략 — 외부 노무사 의존 없이 규칙을 코드로 구현 ② Land & Expand 모듈 확장 전략 ③ 셀프서브 온보딩으로 CAC 최소화 |
| **피해야 할 전략** | "올인원 HR SaaS"로 포지셔닝하면 flex와 정면 충돌. 기능 수에서 절대 승리 불가 |
| **공격 가능 영역** | flex가 구조적으로 진입하기 어려운 "현장(제조/건설/물류)" + "안전관리" + "법적 리스크 통합". flex의 고객은 IT/서비스업 → 제조/건설은 아직 미개발 |

## 2) 시프티 (Shiftee)

### 1. 기업 개요

| 항목 | 내용 |
|------|------|
| **기업 유형** | B2B 근태관리(WFM) 특화 SaaS |
| **주요 고객** | 스타트업~대기업 (SK, 롯데, 카카오 포함), 교대/유연 근무 사업장 중심 |
| **핵심 수익원** | 사용자 수 기반 월 구독료 |

### 2. 가치사슬 분석

#### 2-1. 주요활동

| 활동 | 내용 |
|------|------|
| **투입/조달** | **핵심 자산:** 근태 정밀 계산 알고리즘 (교대·유연·탄력 근무 대응), 위치/WiFi 인증 기술. **확보:** 100% 자체 개발. **경쟁우위:** 복잡한 교대 패턴 처리 정밀도 업계 최고 수준 |
| **운영/생산** | **방식:** 완전 자동화 SaaS. 장비 없이 스마트폰만으로 출퇴근 인증. **비용:** 극도로 린(Lean)한 구조 — 11억 투자로 흑자 달성. 영업이익률 74.5%로 SaaS 업계 최상위 수익성 |
| **산출/배송** | 웹+모바일 앱. 별도 하드웨어 불필요 (위치/WiFi 기반). 관리자와 사용자 모두 모바일 중심 경험 |
| **마케팅/영업** | 제품 주도 성장(PLG), 구전, SEO. 최소 영업 인력. 재구매율 97.2%로 자연 유지. 마케팅 비용 극소화 |
| **서비스** | 셀프서브 기반 CS. 대기업 고객은 전담 지원. 기능 단순→교육 비용 低 |

#### 2-2. 지원활동

| 활동 | 내용 |
|------|------|
| **기업 인프라** | 소규모 린 조직 (~50명 추정). 비즈니스온 인수 → 재독립 과정에서 자생력 입증 |
| **인적 자원** | 개발 중심. 영업/마케팅 인력 최소. "적은 인원으로 높은 수익" 모델 |
| **기술 개발** | 근태 스케줄링 알고리즘이 핵심 IP. 글로벌(대만) 다국적 근무 패턴 대응 |
| **조달/구매** | 클라우드 인프라 비용 극소. 외부 의존도 최저 수준 |

### 3. 핵심 수익창출 포인트

> **"복잡한 근태 패턴의 정밀 처리"가 돈을 버는 지점이다.**

시프티는 "근태 한 가지"를 극도로 잘한다. 3교대, 유연근무, 탄력근무, 선택근무 등 한국 근로기준법의 복잡한 근무 유형을 정확히 계산하고, 장비 없이 스마트폰만으로 출퇴근을 인증한다. 이 "단일 기능의 깊이"가 97.2% 재구매율의 원천이다.

### 4. 경쟁우위 및 약점

| 강점 | 약점 |
|------|------|
| 영업이익률 74.5% — SaaS 업계 최상위 수익성 | 근태 외 기능 확장이 느림 |
| 장비 없는 출퇴근 인증 → 도입 장벽 제로 | EHS/안전관리 역량 전무 |
| 대기업(SK, 롯데) 레퍼런스 확보 | 제조/건설 현장 특화 UX 부재 |
| 97.2% 재구매율 → 자연 유지 수익 | 근태 한 가지에 과도 집중 → 성장 한계 |

### 5. 우리 사업에 대한 시사점

| 구분 | 내용 |
|------|------|
| **배울 점** | ① 린(Lean) 경영 — 11억 투자로 흑자 달성한 비용 구조 ② "잘 하는 것 하나"에 집중하는 버티컬 전략 ③ 장비 불필요 모바일 인증의 도입 용이성 |
| **피해야 할 전략** | 근태관리 정밀도에서 시프티와 경쟁하는 것은 비효율. 근태는 "충분히 좋은 수준"으로 구현하되 차별화 축으로 삼지 말 것 |
| **공격 가능 영역** | 시프티 고객 중 제조/건설/물류 기업은 "근태는 되는데 안전관리가 안 된다"의 Pain을 가짐. 이 고객이 우리의 확장 타겟 |

## 3) 세이프로 (SAFE-PRO) / 노버스메이

### 1. 기업 개요

| 항목 | 내용 |
|------|------|
| **기업 유형** | B2B 기업 리소스 관리 플랫폼 (EHS + 근태 + 자산) |
| **주요 고객** | 제조/건설/공공 사업장 (중소~대기업), 고객사 4만+, 사용자 33만+ |
| **핵심 수익원** | SaaS 구독료 + On-Premise 구축 프로젝트비 |

### 2. 가치사슬 분석

#### 2-1. 주요활동

**① 투입/조달 물류**

| 요소 | 내용 |
|------|------|
| **핵심 자산** | 산업안전보건법·중대재해처벌법 도메인 지식, 업종별 위험성평가 템플릿, 현장 안전 프로세스 노하우 |
| **확보 방식** | 혼합 — 자체 도메인 전문가 + 고객 현장 피드백으로 지속 고도화 |
| **경쟁우위** | 2014년부터 10년+ 축적된 현장 안전관리 데이터·프로세스. 4만+ 사업장에서 검증된 실무 적합성 |

**② 운영/생산**

| 요소 | 내용 |
|------|------|
| **서비스 제공 방식** | SaaS(셀프서브) + On-Premise(프로젝트) 이원 구조. 중소기업은 SaaS, 대기업·공공은 구축형 |
| **핵심 프로세스** | 위험성평가 → PTW(작업허가서) → TBM(작업전 안전점검) → 시설점검 → 사고기록 → 교육관리. 별도로 타임키퍼(근태), 샘큐(자산) 병행 |
| **비용 구조** | SaaS 매출은 한계비용 低, 구축형은 인력 투입 기반 → 인건비 변동비 비중 高. 이원 모델의 비용 복잡성 존재 |

**③ 산출/배송**

| 요소 | 내용 |
|------|------|
| **고객 전달** | 모바일 앱(현장 작업자) + PC 웹(관리자) 100% 연동. On-Premise는 고객사 서버 구축 |
| **사용자 경험** | 현장 작업자 중심 모바일 UX 강조. SOS 기능, 위치·층고 확인 등 현장 특화 기능 |

**④ 마케팅 및 영업**

| 요소 | 내용 |
|------|------|
| **주요 채널** | 정부 조달 등록(나라장터), 산업단지 네트워크, 전시회·세미나, 공공기관 입찰 |
| **세일즈 방식** | B2B 직접 영업 (공공/대기업) + 셀프서브 (중소기업 SaaS). 이원 세일즈 구조 |
| **CAC 구조** | 공공/대기업은 CAC 高 (입찰·제안서·PoC). 중소기업 SaaS는 CAC 中 (온라인·조달) |

**⑤ 서비스**

| 요소 | 내용 |
|------|------|
| **고객 지원** | 구축형 고객은 전담 PM + 커스터마이징 지원. SaaS 고객은 헬프데스크 + 교육 동영상 |
| **유지/업셀** | 세이프로(EHS) → 타임키퍼(근태) → 샘큐(자산) 교차판매. 구축형은 연간 유지보수 계약 |

#### 2-2. 지원활동

| 활동 | 내용 |
|------|------|
| **기업 인프라** | ~100명 규모 추정. 국내+베트남 지사 운영. SaaS/구축형 이원 조직 구조 |
| **인적 자원** | 개발(국내+베트남 오프쇼어) + 안전관리 도메인 전문가 + 공공 영업 인력 혼합 |
| **기술 개발** | EHS 핵심 로직 자체 개발. 모바일 현장 UX에 높은 투자. 커스텀 모듈 개발 역량 보유 |
| **조달/구매** | 베트남 오프쇼어 → 개발 인건비 절감. 클라우드(SaaS) + 서버(On-Premise) 이원 인프라 비용 |

### 3. 핵심 수익창출 포인트

> **"현장 안전 프로세스의 디지털화"와 "구축형 프로젝트의 고단가 수익"이 돈을 버는 두 축이다.**

세이프로의 가치는 종이·엑셀 기반의 안전관리 서류(위험성평가, TBM, PTW 등)를 모바일로 전환하여 **"현장 작업자가 실제로 쓸 수 있게 만든 것"**에 있다. SaaS는 다수 중소기업에서 안정적 반복 수익을, 구축형은 소수 대기업에서 고단가 프로젝트 수익을 만든다.

### 4. 경쟁우위 및 약점

| 강점 | 약점 |
|------|------|
| 4만+ 고객사 / 33만+ 사용자의 시장 검증 | 급여 계산·근로계약 관리 등 HR 핵심 기능 부재 |
| EHS + 근태 + 자산을 별도 제품으로 보유 (통합 가능성) | 3개 제품이 통합되지 않은 별도 플랫폼 (데이터 연결 약함) |
| 현장 작업자 중심 모바일 UX 경쟁력 | SaaS/구축형 이원 모델의 조직·비용 복잡성 |
| 정부 조달 등록 → 공공 시장 접근 유리 | "법적 컴플라이언스 자동화" 수준은 기초 (규칙엔진 약함) |

### 5. 우리 사업에 대한 시사점

| 구분 | 내용 |
|------|------|
| **배울 점** | ① 현장 작업자 UX 설계 — SOS, 위치 확인 등 현장 필수 기능 ② 정부 조달 채널 확보 전략 ③ 베트남 오프쇼어 활용 비용 최적화 |
| **피해야 할 전략** | 구축형(On-Premise)을 초기부터 병행하면 인력·비용 분산. MVP 단계에서는 SaaS Only 집중 |
| **공격 가능 영역** | 세이프로 고객 중 "급여·근로계약도 통합 관리하고 싶다"는 니즈. 세이프로가 타임키퍼와 세이프로를 연결하지 못하는 "데이터 단절" 지점이 공격 포인트 |

|------|------|
| 원/하청 통합 관리라는 명확한 니치 | HR 기능 전무 → 확장성 제한 |
| 시니어·외국인 근로자 대응 UX | 기업 정보·재무 불투명 → 사업 지속성 리스크 |
| "EASY + SAFE" 양면 가치 제안 | 시장 인지도 극히 낮음 |

### 5. 우리 사업에 대한 시사점

| 구분 | 내용 |
|------|------|
| **배울 점** | ① 원/하청 "도급사 법적 방어"라는 Pain의 명확한 정의 ② 시니어·외국인 대응 UX 설계 원칙 |
| **피해야 할 전략** | 원/하청 구조에만 특화하면 시장이 지나치게 좁아짐. 니치에서 시작하되 확장 경로 필수 |
| **공격 가능 영역** | 이지세이프 고객이 "근태·급여도 관리하고 싶다"는 확장 니즈 발생 시 대체 가능 |

# 시장 내 가치사슬 구조 유형 3가지 정의

연구 대상 6개 기업을 분석한 결과, Workforce Risk Management 시장에는 **3가지 뚜렷한 가치사슬 구조 유형**이 존재한다:

---

## 유형 A: "소프트웨어 자동화형" (Software Automation Model)

> **대표 기업:** flex, 시프티

```
[법령/도메인 지식] → [규칙 엔진 코드화] → [SaaS 자동 처리] → [고객 셀프서브]
     (투입)              (운영)              (산출)           (서비스)
```

| 특성 | 설명 |
|------|------|
| **가치 원천** | 복잡한 법률/규칙을 소프트웨어로 자동화. "사람이 할 일을 소프트웨어가 대신" |
| **비용 구조** | 고정비 중심 (개발 인건비). 한계비용 ≈ 0원. 규모의 경제 극대화 |
| **확장 방식** | 기능 모듈 추가 (Land & Expand). 수평적 확장 |
| **약점** | "현장"을 모름. 모든 가치가 사무실 안에서 완결됨. 물리적 현장 접점 제로 |
| **수익 구조** | 인당/사업장 구독료. MRR/ARR 기반 반복 수익. LTV 높음 |

---

## 유형 B: "현장 디지털화형" (Field Digitization Model)

> **대표 기업:** 세이프로, 이지세이프, 스마플

```
[현장 프로세스 지식] → [모바일 앱 개발] → [현장 작업자 사용] → [데이터 축적]
      (투입)              (운영)             (산출)             (서비스)
```

| 특성 | 설명 |
|------|------|
| **가치 원천** | 종이/수기 기반의 현장 안전관리를 모바일로 전환. "현장의 아날로그를 디지털로" |
| **비용 구조** | 개발비 + 현장 커스터마이징 인력비. 구축형 병행 시 변동비 상승 |
| **확장 방식** | 현장 수(사업장) 확대 + 추가 플랫폼 교차판매. 수직적 심화 |
| **약점** | 급여·인사 등 "사무실 업무"를 모름. 모든 가치가 현장에서 완결되고 본사 HR과 연결 안 됨 |
| **수익 구조** | SaaS 구독 + 구축 프로젝트. 혼합 수익 모델 |

---

## 유형 C: "기술 도구형" (Technology Tool Model)

> **대표 기업:** 안전하이

```
[AI 기술] → [CCTV 영상 분석] → [위험 탐지/알림] → [대형 현장 적용]
  (투입)        (운영)             (산출)           (서비스)
```

| 특성 | 설명 |
|------|------|
| **가치 원천** | 첨단 기술(AI, IoT)로 인간의 물리적 한계를 초월. "사람의 눈이 못 보는 것을 기술이 봄" |
| **비용 구조** | R&D 비용 + GPU 인프라 비용 높음. 현장별 커스터마이징 비용 가변적 |
| **확장 방식** | AI 모델 성능 향상 + 적용 현장 확대. 기술 깊이 우선 |
| **약점** | "관리"를 모름. 탐지는 하지만 후속 조치(교육, 서류, 법적 대응)는 빠져 있음. 중소기업에 과도한 스펙 |
| **수익 구조** | 기술 라이선스 + 프로젝트 구축비. 대형 고객 소수 의존 |

### 유형 비교 매트릭스

| 차원 | 유형 A (소프트웨어 자동화) | 유형 B (현장 디지털화) | 유형 C (기술 도구) |
|------|:---:|:---:|:---:|
| **가치 발생 장소** | 사무실 (본사 HR) | 현장 (작업장) | 현장 (CCTV/센서) |
| **고객 접점** | 관리자 (PC 중심) | 작업자 (모바일 중심) | 관제실 (대시보드) |
| **한계비용** | ≈ 0원 | 중간 | 높음 |
| **확장 속도** | 빠름 (셀프서브) | 중간 (현장 온보딩 필요) | 느림 (커스터마이징) |

| **EHS 커버** | ❌ 전무 | ✅ 강함 | ⚠️ 탐지만 가능 |
| **데이터 연결** | HR 데이터만 | EHS 데이터만 | 영상/센서 데이터만 |

# 우리가 들어갈 수 있는 가치사슬 포지션

## 제안: "유형 A + 유형 B의 연결자" (Bridge Model)

현재 시장의 구조적 문제는 **유형 A(사무실 HR)와 유형 B(현장 EHS)가 완전히 단절**되어 있다는 것이다. 어떤 기업도 이 두 유형을 연결하지 않고 있다.

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                  │
│   유형 A의 가치                    유형 B의 가치                │
│   (사무실 HR)                      (현장 EHS)                   │
│                                                                  │
│   급여 계산 ─────┐          ┌───── 위험성평가                    │
│   근태 관리 ─────┤          ├───── 안전교육                      │
│   근로계약 ─────┤          ├───── 사고기록                      │
│   연차 관리 ─────┤          ├───── 법정점검                      │
│                   │          │                                    │
│                   │   ???    │                                    │
│                   │          │                                    │
│                   └──── ❌ ──┘                                    │
│                     단절 지점                                     │
│                   (아무도 연결하지 않음)                          │
│                                                                  │
│                        ▼ ▼ ▼                                     │
│                                                                  │
│                   ★ 우리의 포지션 ★                             │
│                                                                  │
│                   [리스크 통합 분석 레이어]                       │
│                                                                  │
│   급여 데이터 ────┐          ┌───── 안전 데이터                  │
│   근태 데이터 ────┼── 연결 ──┤                                   │
│   계약 데이터 ────┤     ↕    ├───── 사고 데이터                  │
│                   │  리스크  │                                    │
│                   │ 프로파일 │                                    │
│                   └─────────┘                                    │
│                        │                                         │
│                        ▼                                         │
│               [법적 대응 리포트]                                 │
│               [리스크 예측 알림]                                  │
│               [전문가 연결 생태계]                                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 구체적 포지션 정의

| 차원 | 우리의 포지션 |
|------|-------------|
| **투입/조달** | 노동법(유형 A 자산) + 산업안전보건법(유형 B 자산) **모두** 내재화. 노무사 파트너로 법률 지식 코드화 |
| **운영/생산** | "충분히 좋은" HR(급여·근태) + "충분히 좋은" EHS(교육·점검·사고) + **"유일한" 리스크 통합 분석** |
| **산출/배송** | 관리자용 PC 대시보드 + 현장용 모바일 앱 **동시 제공**. 데이터는 양쪽에서 실시간 연결 |
| **마케팅/영업** | 노무사·안전관리자 = 채널. "무료 리스크 진단" = 리드. 산업단지·정부사업 = 네트워크 |
| **서비스** | 전문가 마켓플레이스(노무사·안전관리자가 제품 위에서 서비스) + 법적 대응 리포트 자동 생성 |

### 핵심 차별화: "데이터 연결의 가치"

경쟁사 누구도 제공하지 않는 가치:

| 데이터 연결 시나리오 | 가치 |
|---------------------|------|
| 근태 이상(잦은 결근) + 현장 아차사고 이력 → **고위험 직원 식별** | 사고 사전 예방 |
| 초과근무 데이터 + 사고 발생 시간대 → **피로 기반 리스크 예측** | 작업 배치 최적화 |
| 안전교육 미이수 + 위험 작업 배정 → **법적 위반 자동 경고** | 과태료 사전 방지 |
| 근로계약 만료 + 안전인증 만료 → **행정 기한 통합 관리** | 관리 업무 70% 감소 |
| 전체 데이터 → **노동청 점검·산재 대응 리포트 자동 생성** | 법적 대응 비용 절감 |

# 가장 취약한 경쟁사 유형 1개 도출

## 결론: **"유형 B — 단일 기능 EHS SaaS" (이지세이프 유형)이 가장 취약하다**

### 취약성 근거

| # | 취약 요인 | 구체적 설명 |
|---|-----------|------------|
| 1 | **기능 범위의 좁음** | 안전 점검·위험성평가만 제공. HR(급여·근태) 미지원 → 고객은 결국 추가 솔루션 필요 → "하나로 되면 좋겠다"는 전환 욕구 발생 |
| 2 | **전환 비용 극저** | 축적 데이터가 점검 기록뿐 → 다른 솔루션으로 전환 시 잃을 것이 거의 없음 |
| 3 | **사업 지속성 리스크** | 기업 정보·재무 비공개. 투자·매출 불투명 → 고객의 "이 회사가 내년에도 있을까?" 우려 |
| 4 | **네트워크 효과 부재** | 단일 사업장에서 단독 사용. 원/하청 연결이 있더라도 그 너머의 확장성 없음 |
| 5 | **가격 압박 취약** | 기능이 좁아서 "비싸다" 인식 쉬움. 무료 정부 컨설팅과의 비교에 취약 |

### 공략 전략

```
이지세이프 유형의 고객이 느끼는 Pain:
"안전관리 앱은 쓰고 있는데, 근태·급여는 여전히 엑셀이다"
                          │
                          ▼
우리의 제안:
"안전관리 + 근태 + 급여를 하나에서. 데이터도 연결됩니다"
                          │
                          ▼
전환 조건:
✅ 기존 점검 데이터 마이그레이션 지원
✅ 월 비용 동등 또는 저렴
✅ 추가 기능(급여·근태) 가치만큼의 Premium 정당화
```

### 가장 취약하지 않은 유형 (방어력 최강)

반대로 **가장 방어력이 강한 유형은 "유형 A — flex 유형"**이다.

| 방어 요인 | 설명 |
|-----------|------|
| 급여 데이터 축적 → 전환 비용 극高 | 3년치 급여 이력은 타 시스템으로 이전 극난 |
| Land & Expand로 모듈 깊이 증가 → Lock-in 강화 | 모듈 5개 이상 사용 고객의 이탈률 ≈ 0% |
| 206명 개발 조직의 속도 | 경쟁사가 추격하기 어려운 기능 확장 속도 |
| 기업가치 5,000억의 자본 방어벽 | 가격 전쟁에서도 버틸 여력 확보 |

> **따라서 우리의 전략은 "flex(유형 A)를 공격하지 않고, 이지세이프·안전하이 유형(유형 B/C)의 취약 고객을 흡수"하면서 "flex가 없는 현장 영역"으로 진입하는 것이다.**

## (4) 우리가 채택해야 할 가치사슬 전략

### 전략명: "Bridge SaaS" — 유형 A의 효율 + 유형 B의 현장 접점

HR SaaS(유형 A)의 효율이 최고이지만, 우리가 flex·시프티와 동일한 구조를 취하면 **이미 6년 앞서 있는 경쟁자와 정면 충돌**한다. 따라서:

> **"유형 A의 비용 구조를 취하되, 유형 B의 고객 접점을 확보"하는 하이브리드 전략을 채택한다.**

#### 가치사슬 설계 원칙 (5가지)

| # | 원칙 | 구체적 설계 | 따르는 유형 |
|---|------|-----------|:-----------:|
| 1 | **한계비용 제로를 유지하라** | 구축형(On-Premise) 절대 불가. SaaS Only. 모든 고객이 동일 코드베이스 사용 | 유형 A |
| 2 | **현장 모바일 UX를 확보하라** | 관리자(PC) + 작업자(모바일) 양면 UX. 작업자 앱은 "탭 한 번"으로 전체 기록 완료 | 유형 B |
| 3 | **법령 규칙 엔진을 코드화하라** | 근로기준법 + 산업안전보건법 + 중대재해처벌법을 규칙 엔진으로 내재화. 외부 노무사 의존 최소화 | 유형 A |
| 4 | **데이터 연결을 핵심 가치로 설정하라** | HR 데이터 + EHS 데이터 = 리스크 프로파일. 이것은 어떤 경쟁사도 가지지 않은 유일한 가치 | 신규 |
| 5 | **전문가 생태계를 네트워크 해자로 만들어라** | 노무사·안전관리자가 우리 제품 위에서 서비스 → 추천 채널 + Lock-in + 네트워크 효과 | 신규 |

# 보강 2: 핵심 수익 활동 추출 및 "가장 돈 되는 구조" 정의

## (1) 경쟁사별 수익 발생 지점 — 가치사슬 내 정확한 위치

각 기업이 "실제로 돈을 버는 활동"을 가치사슬에서 정확히 추출한다. 매출원이 아니라 **"고객이 돈을 내는 이유가 되는 활동"**을 식별한다.

| 기업 | 수익 발생 지점 (가치사슬 위치) | 고객이 돈을 내는 진짜 이유 | 수익 유형 |
|------|:---:|:---:|:---:|
| **flex** | 운영 → 급여 자동 계산 | "급여를 틀리면 소송 → flex가 정확히 계산해 주니까" | 🟢 반복 구독 (인당) |
| **시프티** | 운영 → 근태 패턴 처리 | "3교대 스케줄링이 복잡 → 시프티만 정확히 처리" | 🟢 반복 구독 (인당) |
| **세이프로** | 산출 → 현장 모바일 기록 | "안전 서류가 아직 종이 → 모바일로 현장에서 바로 작성" | 🟡 구독 + 구축 혼합 |
| **스마플** | 투입 → 협회 표준 프로세스 | "협회가 만들었으니 법적으로 안전" | 🟡 구독 (신뢰 프리미엄) |
| **안전하이** | 운영 → AI 영상 분석 | "사람의 눈이 닿지 않는 곳을 AI가 감시" | 🔴 프로젝트 + 라이선스 |
| **이지세이프** | 산출 → 도급사 증빙 기록 | "원청 법적 책임 방어 데이터가 필요" | 🟢 구독 (추정) |
| **더존비즈온** | 운영 → 급여↔회계 자동 연동 | "급여 → 회계 전표가 자동 → 세무사가 편해짐" | 🟡 라이선스 + 유지보수 |
| **노무사** | 운영 → 1:1 전문 상담 | "법적 문제 시 내 편이 되어줄 사람" | 🔴 건별 수임 + 월 고문 |

## 최종: "가장 돈 되는 구조" 정의

### 결론: SaaS Only + 법령 규칙 엔진 + 데이터 연결 = 최적 수익 구조

분석한 모든 유형 중, **"가장 돈이 되는 구조"**는 다음 조건을 동시에 만족하는 모델이다:

```
┌───────────────────────────────────────────────────────────────────┐
│                                                                    │
│              ★ "가장 돈 되는 구조" 공식 ★                        │
│                                                                    │
│   수익 = (법령 규칙 엔진의 깊이) × (자동화율) × (데이터 연결 가치) │
│          ─────────────────────────────────────────────────────────  │
│                             인력 투입 비용                          │
│                                                                    │
│   즉:                                                              │
│   ① 법령 규칙을 코드로 내재화할수록 → 진입장벽 ↑ + 전환비용 ↑     │
│   ② 자동화율이 높을수록 → 한계비용 ↓ + 이익률 ↑                   │
│   ③ 데이터가 연결될수록 → 차별화 ↑ + ARPU ↑                       │
│   ④ 인력 투입이 적을수록 → 비용 ↓ + 확장 속도 ↑                   │
│                                                                    │
│   이 네 변수를 동시에 최적화하는 유일한 구조:                      │
│                                                                    │
│        「SaaS Only + 법령 규칙 엔진 + HR↔EHS 데이터 연결」        │
│                                                                    │
└───────────────────────────────────────────────────────────────────┘
```

### 이 구조가 기존 경쟁사보다 "더 돈이 되는" 이유 5가지

| # | 이유 | 기존 경쟁사와의 차이점 |
|---|------|----------------------|
| 1 | **HR+EHS 통합 → ARPU 2배** | flex는 HR만 과금 (인당 5,000~15,000원). 우리는 HR+EHS 통합이므로 사업장 단위 15~30만 원 과금 가능. 동일 고객에서 2배 수익 |
| 2 | **리스크 통합 분석 → 프리미엄 정당화** | 세이프로는 "기록" 도구. 우리는 "리스크 예측" 도구. 예측 가치는 기록 가치보다 가격 민감도가 낮음 (보험 심리) |
| 3 | **노무사 생태계 → CAC 절감 + 추천 수수료** | 노무사가 추천 → CAC 10만 원 이하 가능. 동시에 노무사에게 추천 수수료(MRR 10~20%) 지불해도 순이익 증가 |
| 4 | **법적 대응 리포트 자동 → 서비스 차별화** | 사고 발생 시 자동 리포트는 "위기 순간의 가치" = 가격 민감도 소멸. 긴급 온보딩 가능 |
| 5 | **데이터 축적 → 전환 비용 → Churn 감소** | HR 데이터 + EHS 데이터가 동시에 축적 → 전환 시 잃을 것이 2배 → Churn 50% 감소 → LTV 2배 |

### 최종 수익 모델 설계

| 수익원 | 과금 방식 | 예상 비중 | ARPU 기여 |
|--------|----------|:---------:|:---------:|
| **기본 구독 (HR+EHS 통합)** | 사업장 월 정액 (15~30만 원) | 70% | 기본 |
| **프리미엄 리포트** | 법적 대응 리포트·리스크 분석 월 추가 과금 | 15% | ARPU +5~10만 원 |
| **전문가 연결 수수료** | 노무사·안전관리자 매칭 시 중개 수수료 | 10% | ARPU +2~5만 원 |
| **데이터 서비스 (장기)** | 익명화된 산업별 리스크 벤치마크 리포트 | 5% | 고객 외 수익 |
